# Gym Aspect

このディレクトリはジムセッションの記録と管理を担当します。

## ディレクトリ構成

| パス | 内容 |
|------|------|
| `logs/YYYY-MM-DD.md` | ジムセッション実績ログ（Notion ジム DB のミラー） |
| `gyms/<chain>/<location>.md` | ジムのマシン一覧・設備情報 |
| `profile.md` | ジム会員情報・個人目標 |
| `../goal.md` | 共有目標（diet と gym が参照） |

## Notion が source of truth（厳守）

**ジムに関する全ての判断・回答・コマンドの最初に Notion → md 同期を実行する。**

```bash
bun run scripts/gym/sync-notion-to-md.ts
```

- `logs/` はあくまで Notion のミラー。ローカルにないセッションが Notion にある場合がある（記録忘れ・他端末からの入力）
- 「最後のジムはいつ？」「今日行ったほうがいい？」など、**事実判断を伴う質問に答える前に必ず同期する**
- スクリプトは差分のみ書き出す冪等動作（最新ローカル日付以降を取得）。コストは最小
- 既存ファイルはデフォルトで上書きしない。Notion 側の変更を取り込みたい場合は `--force`
- FB の生テキスト（"ギリ一回できず"、複数行コメントなど）も保持される

## メニュー生成

静的なメニューファイルは持たない。`/gym plan` 実行時に以下を動的参照してメニューを決定する:

1. **前日のジムセッション確認（厳守）**: `notion-list.ts --date 前日` でエントリーを確認する。**見つからない場合でも「なし」と断定せず、必ず `notion-search` で `ジム M/D`（例: `ジム 4/15`）を再検索すること。** JST/UTC ズレで `notion-list.ts` に表示されないケースがある
2. `logs/` の直近3日のログ → 前回の種目・重量・フィードバック
3. `gyms/<location>.md` → 利用可能なマシン一覧
3. `../goal.md` → 現在の目標（重量・体組成）
4. `../daily/diary/YYYY-MM-DD.md` → 当日のコンディション（`## コンディション` セクション）
   - `low` → 全体的に軽め。重量・セット数を落とす
   - `normal` → 通常プラン
   - `high` → 重量・セット数を増やしてしっかり攻める
   - セクションなし → `normal` 扱い

## 重量調整ルール（厳守）

`/gym plan` 実行時、前回ログのフィードバックに基づいて**必ず**重量を見直す:

- `余裕` → 次回は +5kg（または回数を増やす）
- `まあまあ` → 現状維持
- `無理` / `きつい` → 次回は -5kg
- **フィードバック未記入の種目はデフォルトで `余裕` 扱いとする**（毎回記入不要）

全種目を横断して確認すること。一種目でも見落としたら失敗。

---

## メニュー密度（厳守）

連日ルールで制限がある場合でも、**使える器具を全て洗い出してから**メニューを組む。
制約を確認 → 使える器具リスト作成 → その中から3〜5種目を選定。
「制限があるから少なめ」は禁止。使えるものは積極的に使う。

---

## 補助種目のローテーション（厳守）

メニュー決定時、**直近3セッション分の Notion ジム DB を確認し**、補助種目が重複しないよう選ぶ。

- **対象**: メイン種目（ダンベルプレス・フィックスドプルダウン・スクワットマシン・ウォーキング）以外の全種目
- **ルール**: 直近3セッション以内に使った補助種目は原則使わない
- **選択肢**: Seated Leg Curl / Prone Leg Curl / Leg Extension / Inner・Outer Thigh / Shoulder Press / Low Row / Lat Pull Down / Abdominal / Standing Multi Flight / Plate Loaded Shoulder Press など、その日の部位バランスに応じて選ぶ
- **バリエーション重視**: 同じマシンばかり使うと筋肉が適応して成長が鈍化する。毎回なるべく異なる器具を選び、幅広く使い回すこと

例: 前回・前々回でアブドミナルを使っていたら → Seated Leg Curl や Shoulder Press を選ぶ

## 種目プリファレンス（厳守）

- **胸**: ダンベルプレスを使う（ベンチプレスはケガリスクがあるため使わない）
- **脚**: スクワットマシンを使う（フリーウエイトスクワットは使わない）
- **デッドリフト**: 使わない
- **基本方針**: BIG3（ベンチプレス・スクワット・デッドリフト）にこだわらない。マシン優先で組む

## Notion エントリ構造

**1セッション = 1ページ。** 種目データはプロパティではなくページ本文のテーブルに記載する。

- ページプロパティ: `名前: ジム`、`日付`（開始〜終了）のみ
- ページ本文: `scripts/gym/format-menu.ts` で生成したスタイリング済みテーブル
  - 📊 セッションサマリー（青 callout）
  - 💪 筋トレテーブル（青ヘッダー: FB, 種目, 重量（kg）, セット, 回数）
  - 🏃 有酸素テーブル（緑ヘッダー: FB, 種目, 時間, 傾斜, スピード）
  - **FB は1列目（左端）** — 横スクロールせず編集できるようにするため

**FB（フィードバック）は3択:** `余裕` / `まあまあ` / `きつい`（自由テキストは `normalizeFeedback()` で正規化）

## ログフォーマット

`logs/YYYY-MM-DD.md`:

```markdown
# ジムログ YYYY-MM-DD

## 種目名
- 重量: Xkg × Y回 × Zセット
- FB: 余裕

メモ: （体感メモ）
```
