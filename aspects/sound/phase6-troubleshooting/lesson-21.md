## 1. ハウリングとは何か（10分）

### フィードバックループの仕組み

ハウリングは「音のループ」です。以下の図を見てください。

```
  ┌─────────────────────────────────────────────────────┐
  │                                                     │
  ▼                                                     │
 [マイク] ──→ [ミキサー] ──→ [アンプ] ──→ [スピーカー] ─┘
  拾う        増幅する       さらに増幅    音を出す
                                           ↓
                                      その音がマイクに
                                      戻ってしまう！
```

**流れ:**

```
1. マイクが声を拾う
2. ミキサーで増幅する
3. スピーカーから音が出る
4. その音がマイクに入る     ← ここが問題
5. また増幅される
6. またスピーカーから出る
7. さらにマイクに入る...
8. 無限ループ → キーーーン！
```

### なぜ特定の周波数で起きるのか

ハウリングは全帯域で一斉に起きるわけではありません。部屋の形、スピーカーの特性、マイクの特性が組み合わさって、**特定の周波数だけ**がループ条件を満たします。

```
  音量
  ▲
  │          ×  ← この周波数だけループ条件を満たす
  │    ──────×──────── フィードバック閾値
  │   /      ×       \
  │  / ×    ×  ×      \
  │ /×  ×  ×    ×  ×   \
  │/  ×  ×        ×  ×  \
  ├────────────────────────→ 周波数
  低                     高

  × = マイク→スピーカー→マイクで戻ってくる音の大きさ
```

よくハウリングが起きる周波数帯:
- **2kHz - 4kHz**: 人の耳に最も敏感な帯域。キーンという音
- **500Hz - 1kHz**: ウォーンという低めのハウリング
- **250Hz 以下**: ボーンと部屋全体が共鳴する

> **田中メモ:** 配信では会場のスピーカー音がマイクに回り込むだけでなく、配信用のモニタースピーカーから回り込むパターンもある。イヤホンモニターを使えば配信側のフィードバックリスクはゼロにできる。

## 2. GBF（Gain Before Feedback）の概念（5分）

**GBF = フィードバックが起きる直前まで、どれだけゲインを上げられるか**

これが PA オペレーターの腕の見せどころです。

```
  音量
  ▲
  │ ─ ─ ─ ─ ─ ─ ─ ─ ─  フィードバック発生ライン
  │
  │ ════════════════════  GBF（ここまで上げられる）
  │ ░░░░░░░░░░░░░░░░░░░  安全マージン（6dB）
  │ ════════════════════  実用レベル（ここで運用する）
  │
  ├────────────────────→
```

**GBF を高くする = より大きな音量で使える = ハウリングしにくい**

GBF を高くする方法が、次のセクションで学ぶ「予防策」です。

## 3. 予防策 5つ（15分）

### 予防策 1: マイクとスピーカーの位置関係

**鉄則: マイクはスピーカーの後ろか横に置く。絶対に正面に置かない。**

```
  ◎ = スピーカー    🎤 = マイク

  【正しい配置】                【絶対NG】

      ◎ → → → → 会衆席           ◎ → → → →
      ↑                           ↓
     🎤（スピーカーの後ろ）       🎤（スピーカーの前）
                                   = 直接音がマイクに入る！
```

教会の典型的な配置:

```
  ┌──────────────────────────────────┐
  │           会衆席                  │
  │                                  │
  │    ◎ メインL        メインR ◎    │
  │     ↓                  ↓        │
  │  ──────── ステージ ────────      │
  │                                  │
  │  🎤 ← 講壇マイク                │
  │     （メインスピーカーより後方）   │
  │                                  │
  │  🎤🎤🎤🎤 ← ワーシップチーム      │
  │     （メインスピーカーより後方）   │
  └──────────────────────────────────┘
```

### 予防策 2: モニターの角度と配置

フロアモニターは**マイクの真正面**を避けて角度をつけて置きます。

```
  【良い配置】              【悪い配置】

    🎤                        🎤
    │  ＼                     │
    │    ＼                   │
    │     🔊                  🔊
   口の方向  斜め45°          真正面 = 危険！
```

> **田中メモ:** うちの教会はフロアモニターが4台。ワーシップ中にリーダーがモニターの前を横切ると一瞬ハウリングしかける。モニター位置をテープで床にマーキングしておくと、チームにも伝わりやすい。

### 予防策 3: ゲイン設定を適切に

Phase 2 で学んだゲイン設定の復習です。

```
  ゲインの段階:

  過大 ██████████████████████ ← ハウリングの原因！
  適正 ██████████████         ← ピーク時に -6dB 程度
  過小 █████                  ← 聞こえない → フェーダー上げすぎ → 結局ハウリング

  ※ ゲインを低く設定 → フェーダーを大幅に上げる → ノイズ増加 → 悪循環
```

### 予防策 4: EQ で問題周波数をカット

ハウリングしやすい周波数がわかっているなら、あらかじめ EQ で少し下げておきます。

```
  EQ カーブ（ボーカルマイク例）:

  +6dB ┤
       │
   0dB ┤────────╲──────────────────
       │         ╲    ╱
  -6dB ┤          ╲╱     ← 3kHz付近を -3dB カット
       │               （ハウリングしやすい帯域）
 -12dB ┤
       ├──┬──┬──┬──┬──┬──┬──┬──→ Hz
       80 160 500 1k 2k 4k 8k 16k
```

### 予防策 5: ハイパスフィルター（HPF）

ボーカルマイクには**必ず HPF をオン**にします。

```
  HPF の効果:

  +6dB ┤
       │
   0dB ┤         ╱──────────────── ← 80Hz以上は通す
       │        ╱
  -12dB┤      ╱    ← 80Hz以下をカット
       │    ╱       （低域のハウリング防止 + 不要なノイズ除去）
  -24dB┤  ╱
       ├──┬──┬──┬──┬──┬──┬──→ Hz
       20 50 80 200 500 1k 4k
```

低域のハウリングは「ボーン」という鳴り方で、発見が遅れがち。HPF で予防しましょう。

## 4. モニターのリンギングアウト手順（10分）

リハーサル前に行う「モニターのハウリング周波数を潰す」作業です。

### 手順

```
1. 会場を静かにする
2. モニターに送るマイクのフェーダーを 0dB にする
3. モニターの AUX センドをゆっくり上げていく
   │
   ▼
4. 「キーン」「ウォーン」と鳴り始めたら → その周波数を特定
   │
   ▼
5. パラメトリック EQ でその周波数をナローカット（Q=高め、-6dB程度）
   │
   ▼
6. もう一度 AUX センドを上げていく
   │
   ▼
7. 次の周波数が鳴り始めたら → 同じようにカット
   │
   ▼
8. 3-4箇所カットしたら終了（やりすぎると音質劣化）
```

**EQ カットのイメージ:**

```
  ナローカット（Q = 10-15）

   0dB ┤──────╲──────╱──────
       │       ╲    ╱
  -6dB ┤        ╲╱
       │     ←狭い→
       ├──────────────────→ Hz
             問題の周波数
```

> **田中メモ:** デジタルミキサーなら各モニターバスに個別の EQ がある。アナログ時代は 31 バンドのグラフィック EQ を繋いでたけど、今はミキサー内蔵の PEQ で十分。リンギングアウトの結果はシーン保存しておこう。

## 5. 礼拝中の即時対処（10分）

**ハウリングが起きたら、考える前に手を動かす。**

### 3ステップ対処法

```
  ┌─────────────────────────────────────┐
  │ STEP 1: フェーダーを下げる（1秒）    │
  │   → 疑わしいチャンネルを即座に下げる │
  ├─────────────────────────────────────┤
  │ STEP 2: 原因を特定する（5秒）        │
  │   → どのマイク？ メイン？ モニター？  │
  ├─────────────────────────────────────┤
  │ STEP 3: 修正して戻す（10秒）         │
  │   → ゲイン調整 or EQ → ゆっくり戻す │
  └─────────────────────────────────────┘
```

### どのフェーダーを下げるべきか

```
  判断フローチャート:

  ハウリング発生！
       │
       ▼
  メインスピーカーから鳴ってる？
       │
    ┌──┴──┐
    Yes    No → モニターから鳴ってる
    │           │
    ▼           ▼
  マスター      該当モニターの
  フェーダー    AUX センドを下げる
  を少し下げる
       │
       ▼
  各チャンネルを1本ずつミュート
  → 鳴り止んだチャンネルが犯人
```

### よくある教会でのシナリオ

**シナリオ 1: ワーシップリーダーがモニターの前を歩く**
```
  対処: リーダーのボーカルマイクのフェーダーを素早く下げる
  予防: 立ち位置をリハで決めておく。テープでマーク
```

**シナリオ 2: 誰かがフロアモニターの音量を上げすぎる**
```
  対処: 該当モニターの AUX マスターを下げる
  予防: モニター音量はオペレーターが管理。勝手に触らせない
```

**シナリオ 3: 説教マイクがメインスピーカーに近すぎる**
```
  対処: 説教マイクのフェーダーを下げて EQ 調整
  予防: 講壇の位置を固定。スピーカーとの距離を確保
```

## 6. フィードバック抑制機（FBD）について（5分）

市販のフィードバックサプレッサーやデジタルミキサー内蔵の FBS 機能について。

```
  仕組み:
  ┌──────────┐    ┌──────────────┐    ┌──────────┐
  │ マイク入力 │ → │ FBD が監視    │ → │ 出力      │
  │          │    │ ハウリング検出 │    │          │
  └──────────┘    │ → 自動ノッチ  │    └──────────┘
                  └──────────────┘
```

**メリット:**
- 自動でハウリング周波数を検出してカットしてくれる
- 初心者でも安心

**デメリット・注意点:**
- 音楽の倍音をハウリングと誤検出することがある
- カットしすぎると音質が劣化する
- 過信して GBF の基本を怠ると、根本的な問題は解決しない

> **田中メモ:** FBD は「保険」として使うもの。基本のマイク配置・ゲイン・EQ ができていれば、FBD に頼る場面はほとんどない。配信では会場のハウリングがそのまま乗るので、会場側でしっかり対策するのが最優先。

## 実践ドリル（10分）

1. **フィードバックループ図を紙に描く**: マイク → ミキサー → スピーカー → マイクの流れを自分で描いてみる
2. **教会の配置図を描く**: スピーカーとマイクの位置関係を確認。マイクがスピーカーの前に来ている箇所はないか
3. **ロールプレイ**: 「礼拝中にキーンと鳴った！」→ 3ステップ対処を声に出しながらシミュレーション
4. **HPF チェック**: 次のリハーサルで、全ボーカルマイクの HPF がオンになっているか確認する

## 教会での応用

- **リハーサル時**: 毎回、モニターのリンギングアウトを行う。デジタルミキサーならシーン保存で効率化
- **新しいメンバーが入ったら**: マイクの持ち方とスピーカーとの位置関係を最初に教える
- **季節の変化**: 冬は窓を閉めるので部屋の残響が変わり、ハウリング周波数もずれることがある
- **配信への影響**: 会場でハウリングが起きると配信にもそのまま乗る。配信リスナーはイヤホンで聴いているので、ダメージが大きい。予防が最重要
