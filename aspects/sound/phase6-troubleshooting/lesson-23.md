# Lesson 23: 緊急対応マニュアル --- 礼拝中のトラブルに即対処する

> **Phase 6 / L23: 礼拝中の7つの緊急シナリオに即座に対処できるようになる**
> **担当:** 田中光（配信・録音）
> **所要時間:** 約60分
> **前提:** Phase 1-5 完了

---

## 今日のゴール
- PA オペレーターの緊急対応マインドセットを理解する
- 7つの緊急シナリオに対して、手順を見なくても対処できるようになる
- 緊急対応カードとプリサービスチェックリストを作成する

---

田中です。「画面の向こうにも会衆がいる」。

今日は一番実践的なレッスンです。機材が壊れるのは、いつも「今日に限って」起きます。リハーサルでは完璧だったのに、礼拝本番で突然トラブルが発生する。そのとき、パニックになるか冷静に対処できるかで、礼拝の空気が変わります。

PA オペレーターのトラブル対応は「見えない奉仕」です。会衆が気づかないうちに問題を解決する。それが理想。

---

## 1. 緊急対応のマインドセット（5分）

### PA オペレーターの3原則

```
  ┌──────────────────────────────────────────────┐
  │                                              │
  │   1. 冷静であること（Calm）                    │
  │      → 焦ると判断ミスが起きる                 │
  │                                              │
  │   2. 体系的であること（Methodical）             │
  │      → 闇雲にいじらない。手順に従う            │
  │                                              │
  │   3. 目立たないこと（Invisible）               │
  │      → 会衆に不安を与えない                    │
  │                                              │
  └──────────────────────────────────────────────┘
```

### 対処の優先順位

```
  ┌─────────────────────────────────────────┐
  │  優先度 1: 有害な音を止める             │
  │  （ハウリング、大音量ノイズ、歪み）      │
  │  → すぐにフェーダーを下げる / ミュート   │
  ├─────────────────────────────────────────┤
  │  優先度 2: 音声を復旧する               │
  │  （礼拝の進行を止めない）               │
  │  → バックアップに切り替える              │
  ├─────────────────────────────────────────┤
  │  優先度 3: 根本原因を修正する           │
  │  （礼拝後でもOK）                       │
  │  → 原因調査、機材交換、設定修正         │
  └─────────────────────────────────────────┘

  ※ 優先度 1 は「秒」、優先度 2 は「分」、優先度 3 は「礼拝後」
```

> **田中メモ:** 配信では「数秒の無音」より「数秒のハウリング」のほうがダメージが大きい。判断に迷ったらミュートする。無音は「技術的な問題があったんだな」で済むけど、耳を劈くハウリングは視聴者を失う。

## 2. 緊急シナリオ 7選（30分）

### シナリオ 1: 説教中にハウリング

```
  状況: 牧師が説教している最中にキーンと鳴る

  ┌─────────────────────────────────────────┐
  │ STEP 1 (1秒): 説教マイクのフェーダーを  │
  │   素早く 10dB 下げる（ミュートしない！）  │
  │   → 説教は続けられる状態を維持          │
  ├─────────────────────────────────────────┤
  │ STEP 2 (5秒): ハウリング周波数を推定     │
  │   → 高い音 = 2-4kHz / 低い音 = 200-500Hz│
  ├─────────────────────────────────────────┤
  │ STEP 3 (10秒): PEQ でナローカット       │
  │   → 推定周波数を -6dB、Q = 10-15       │
  ├─────────────────────────────────────────┤
  │ STEP 4 (5秒): フェーダーをゆっくり戻す  │
  │   → 元のレベルの 90% くらいまで          │
  └─────────────────────────────────────────┘

  合計: 約20秒で復旧
```

**ポイント:** 説教マイクは完全ミュートしない。牧師は話し続けているので、急に音が消えると礼拝の流れが途切れる。

### シナリオ 2: 音声が完全に出なくなる

```
  状況: 突然、メインスピーカーから何も聞こえなくなる

  ┌─────────────────────────────────────────┐
  │ STEP 1 (2秒): マスターフェーダーを確認   │
  │   → 誰かが触った？ 0dB にあるか？        │
  ├─────────────────────────────────────────┤
  │ STEP 2 (5秒): メインバスのメーターを確認 │
  │   → メーターが振れてる = ミキサーは正常   │
  │   → メーターが振れてない = 入力側の問題   │
  ├─────────────────────────────────────────┤
  │ STEP 3A (メーター振れてる場合):          │
  │   アンプの電源を確認                     │
  │   → スピーカーケーブルを確認              │
  │   → アンプがプロテクト状態ではないか      │
  ├─────────────────────────────────────────┤
  │ STEP 3B (メーター振れてない場合):        │
  │   各チャンネルのミュートを確認            │
  │   → 全チャンネルミュートされてないか      │
  │   → シーンが切り替わっていないか          │
  └─────────────────────────────────────────┘
```

```
  チェックフロー図:

  音が出ない！
       │
       ▼
  マスターフェーダーは上がってる？ ─ No → 上げる → 解決
       │
      Yes
       │
       ▼
  メインバスのメーターは振れてる？
       │
    ┌──┴──┐
   Yes    No
    │      │
    ▼      ▼
  アンプ確認   チャンネル確認
  ケーブル確認  ミュート確認
  電源確認     シーン確認
```

### シナリオ 3: マイクが1本だけ死ぬ

```
  状況: ワーシップ中にボーカルマイク1本が無音になる

  ┌─────────────────────────────────────────┐
  │ STEP 1 (3秒): そのチャンネルのメーター   │
  │   を確認。信号来てる？                   │
  ├─────────────────────────────────────────┤
  │ STEP 2A (信号なし = ワイヤレスの問題):   │
  │   → 受信機のディスプレイを確認           │
  │   → RF シグナルは出てる？                │
  │   → バッテリー残量は？                   │
  │   → マイクの電源スイッチは ON？          │
  ├─────────────────────────────────────────┤
  │ STEP 2B (信号あり = ミキサー側の問題):   │
  │   → フェーダーは上がってる？             │
  │   → ミュートされてない？                 │
  │   → バスアサインは正しい？               │
  ├─────────────────────────────────────────┤
  │ STEP 3: バックアップマイクに切り替え      │
  │   → 予備のハンドマイクを渡す             │
  │   → 空いてるチャンネルに接続             │
  │   → ゲイン設定 → フェーダーアップ        │
  └─────────────────────────────────────────┘
```

> **田中メモ:** ワイヤレスマイクが死んだとき、一番多い原因は「マイク側の電源スイッチが切れた」か「バッテリー切れ」。ステージ横に予備マイク（有線でもいい）を常にスタンバイしておこう。

### シナリオ 4: 配信の音声が途切れる

```
  状況: 現地は問題ないが、オンライン視聴者から「音が出ない」と連絡

  ┌─────────────────────────────────────────┐
  │ STEP 1 (5秒): 配信バス（AUX/BUS）の     │
  │   メーターを確認。信号来てる？            │
  ├─────────────────────────────────────────┤
  │ STEP 2A (信号なし):                      │
  │   → 配信バスへのセンドを確認             │
  │   → 配信バスのフェーダーを確認           │
  │   → 配信バスがミュートされてないか       │
  ├─────────────────────────────────────────┤
  │ STEP 2B (信号あり = PC/配信側の問題):    │
  │   → 配信ソフト（OBS等）の音声メーターを   │
  │     確認                                │
  │   → USB/オーディオインターフェースの接続  │
  │   → 配信ソフトの音声入力設定を確認       │
  ├─────────────────────────────────────────┤
  │ STEP 3: バックアップ配信に切り替え        │
  │   → サブ PC があればそちらで配信          │
  │   → なければスマートフォンで緊急配信      │
  └─────────────────────────────────────────┘
```

```
  配信音声の信号経路:

  [各チャンネル]
       │
       ▼
  [配信バス (AUX/BUS)] ← ここをまず確認
       │
       ▼
  [USB/オーディオIF]   ← 物理接続を確認
       │
       ▼
  [配信PC (OBS等)]     ← ソフト設定を確認
       │
       ▼
  [配信サーバー]       ← ネット回線を確認
       │
       ▼
  [視聴者]
```

### シナリオ 5: 突然の大きなポップ音・バン音

```
  状況: 「バンッ！」という大きな音がスピーカーから出た

  ┌─────────────────────────────────────────┐
  │ STEP 1 (1秒): 全チャンネルミュート       │
  │   ※ ミキサーの MUTE ALL があれば使う     │
  │   ※ なければマスターフェーダーを下げる    │
  │                                         │
  │   ★ 会衆の安全が最優先！                 │
  ├─────────────────────────────────────────┤
  │ STEP 2 (30秒): 原因を調査               │
  │   → ケーブルがショートしてないか          │
  │   → 機器の電源が落ちて再起動してないか    │
  │   → 誰かがファンタム電源をいきなり         │
  │     ON/OFF してないか                    │
  ├─────────────────────────────────────────┤
  │ STEP 3: 1チャンネルずつ復旧              │
  │   → まず説教/MC マイクだけ ON            │
  │   → 問題なければ楽器を1つずつ ON          │
  │   → ポップ音が再発したチャンネルが犯人    │
  └─────────────────────────────────────────┘
```

**ポイント:** ファンタム電源 (+48V) を入れたままコンデンサーマイクの XLR を抜き差しすると、大きなポップ音が出る。ケーブル交換時はファンタムを切ってから。

### シナリオ 6: 停電

```
  状況: 礼拝中に電源が落ちる

  ┌─────────────────────────────────────────┐
  │ STEP 1: 落ち着く。会衆も暗闇で不安。     │
  │   → 牧師にアコースティック進行を促す      │
  ├─────────────────────────────────────────┤
  │ STEP 2: UPS（無停電電源装置）があれば:   │
  │   → 配信 PC と配信用機器を優先的に維持    │
  │   → ミキサーが UPS に繋がっていれば       │
  │     説教マイクだけでも音を出す            │
  ├─────────────────────────────────────────┤
  │ STEP 3: 電源復旧時の注意:               │
  │   → 全フェーダーを下げてから機器を ON     │
  │   → アンプは最後に ON（突入電流防止）     │
  │   → デジタルミキサーの起動を待つ          │
  │   → 保存していたシーンをリコール          │
  └─────────────────────────────────────────┘
```

```
  電源 ON の順序（復旧時）:

  1. [ミキサー]      ← 最初に ON（起動に時間がかかる）
  2. [エフェクター]   ← ミキサーの次
  3. [配信 PC]       ← 配信ソフトの起動
  4. [パワーアンプ]   ← 最後に ON

  ※ 逆順で OFF にする（アンプが最初に OFF）
```

> **田中メモ:** 停電は滅多にないけど、起きたときの衝撃は大きい。UPS はミキサーと配信 PC だけでもつけておくと安心。5分でも持てば、その間にアコースティック礼拝に切り替えられる。配信は UPS が持つ間だけでも続けられるといい。

### シナリオ 7: 1チャンネルが歪む

```
  状況: ボーカルマイク1本だけ音が歪んでいる

  ┌─────────────────────────────────────────┐
  │ STEP 1 (3秒): そのチャンネルのゲインを   │
  │   5dB 下げる                             │
  │   → 歪みが消えたら = ゲイン過大が原因    │
  ├─────────────────────────────────────────┤
  │ STEP 2 (ゲイン下げても歪む場合):         │
  │   → ケーブルを確認（予備に交換）          │
  │   → マイク本体の問題？（予備マイクで確認）│
  ├─────────────────────────────────────────┤
  │ STEP 3 (チャンネル内のエフェクト確認):   │
  │   → コンプの閾値が低すぎないか            │
  │   → EQ で極端にブーストしてないか         │
  │   → インサートに何か挿さってないか        │
  └─────────────────────────────────────────┘
```

## 4. プリサービスチェックリスト（10分）

**トラブルの大半は事前チェックで防げます。**

```
  ┌────────────────────────────────────────────┐
  │      礼拝前チェックリスト（30分前）         │
  ├────────────────────────────────────────────┤
  │                                            │
  │ □ ミキサーの電源 ON・シーンリコール完了     │
  │ □ メインスピーカーから音が出ることを確認    │
  │ □ サブウーファーの電源 ON                   │
  │                                            │
  │ 【ワイヤレスマイク（6本全て）】              │
  │ □ バッテリー交換済み（新品または満充電）    │
  │ □ 電源 ON → 受信機にシグナルあり            │
  │ □ 実際に声を出してメーター確認              │
  │ □ ミュート解除されている                   │
  │                                            │
  │ 【楽器チャンネル】                          │
  │ □ キーボード 1（ステレオ）: 音出し確認      │
  │ □ キーボード 2（ステレオ）: 音出し確認      │
  │ □ アコースティックギター: 音出し確認        │
  │ □ エレキギター: 音出し確認                  │
  │ □ ベース: 音出し確認                        │
  │ □ ドラム: 各マイク音出し確認                │
  │                                            │
  │ 【その他入力】                              │
  │ □ iPhone（BGM）: テスト再生 → 音出し確認    │
  │ □ PC（カウントダウン）: テスト再生確認      │
  │                                            │
  │ 【モニター】                                │
  │ □ 各フロアモニターから音が出ることを確認    │
  │ □ モニターミックスのバランスを確認          │
  │                                            │
  │ 【配信】                                    │
  │ □ 配信バスに信号が来ている                  │
  │ □ PC の配信ソフトが音声を受け取っている     │
  │ □ テスト配信 → 別端末で音声確認             │
  │ □ 録画も開始されている                      │
  │                                            │
  │ 【予備の準備】                              │
  │ □ 予備 XLR ケーブル × 2 をミキサー横に     │
  │ □ 予備バッテリー × 6 をミキサー横に         │
  │ □ 予備ハンドマイク（有線）をスタンバイ      │
  │                                            │
  │ 【最終】                                    │
  │ □ ミキサーのシーンを保存                    │
  │ □ 全体のメーターを確認（クリッピングなし）  │
  │ □ 不要なチャンネルはミュート                │
  │                                            │
  └────────────────────────────────────────────┘
```

> **田中メモ:** このチェックリストを印刷してミキサーに貼っておこう。毎週やれば10分で終わるようになる。「全チャンネルテスト済み」の安心感は礼拝中の精神的余裕に直結する。配信のテスト確認は忘れがちだけど、ここを飛ばすと画面の向こうの会衆が無音礼拝になる。

## 5. ポストインシデントレビュー（5分）

トラブルが起きた後は、必ず振り返りをします。

```
  ポストインシデントレビュー:

  ┌──────────────────────────────────────┐
  │ 1. 何が起きた？                       │
  │    → 事実を正確に記録                 │
  │                                      │
  │ 2. なぜ起きた？                       │
  │    → 根本原因を掘り下げる             │
  │    →「なぜ？」を5回繰り返す           │
  │                                      │
  │ 3. どう対処した？                     │
  │    → 対応の手順と所要時間を記録       │
  │                                      │
  │ 4. 次回どう防ぐ？                     │
  │    → 具体的な再発防止策を決める       │
  │    → チェックリストに追加             │
  └──────────────────────────────────────┘
```

**「なぜ？」を5回繰り返す例:**

```
  問題: 礼拝中にワイヤレスマイクが切れた

  なぜ？ → バッテリーが切れた
  なぜ？ → 古いバッテリーを使っていた
  なぜ？ → 新品と使用済みが混ざっていた
  なぜ？ → 使用済みの回収ボックスがなかった
  なぜ？ → バッテリー管理のルールが決まっていなかった

  → 対策: 新品ボックスと使用済みボックスを設置。
    礼拝前に必ず新品に交換するルールを明文化。
```

## 実践ドリル（10分）

1. **シナリオ暗記テスト**: 7つのシナリオのタイトルを見ないで言えるか。各シナリオの STEP 1 を即答できるか
2. **ロールプレイ**: パートナーに「今、ハウリングが起きた！」「マイクが死んだ！」と言ってもらい、手を動かしてシミュレーション
3. **チェックリスト実践**: 次の日曜日、このチェックリストを使って実際にプリサービスチェックを行う
4. **緊急対応カード作成**: 上のテンプレートを元に、自分の教会用にカスタマイズして印刷・ラミネートする

## 教会での応用

- **チーム全員がカードを理解する**: PA 担当が複数いるなら、全員が同じ対応手順を共有する
- **年2回の緊急訓練**: 実際にケーブルを抜いたり、マイクの電源を切ったりして、対処の練習をする
- **牧師・ワーシップリーダーとの連携**: トラブル時の合図を決めておく（例: 手を挙げたら「マイクに問題あり」）
