## 1. 継続的改善サイクル: PDCA（10分）

### Plan → Do → Check → Act

```
        ┌───────────┐
        │   Plan    │ ← 次の礼拝の目標を決める
        │  計画する  │
        └─────┬─────┘
              │
              ▼
        ┌───────────┐
        │    Do     │ ← 礼拝で実践する
        │  実行する  │
        └─────┬─────┘
              │
              ▼
        ┌───────────┐
        │   Check   │ ← 録音を聴いて振り返る
        │  確認する  │
        └─────┬─────┘
              │
              ▼
        ┌───────────┐
        │    Act    │ ← 改善策を次に反映する
        │  改善する  │
        └─────┬─────┘
              │
              └──────→ Plan に戻る（毎週繰り返す）
```

### 具体的に何をするか

```
  月曜日: Check — 日曜の録音を聴く（30分）
  火曜日: Act   — 改善点をメモに書き出す（15分）
  土曜日: Plan  — 次の礼拝の目標を1つ決める（10分）
  日曜日: Do    — 礼拝で実践する
```

**目標の例:**

```
  週 1: 「ボーカルの子音を聞き取りやすくする」
         → プレゼンス帯（3-5kHz）を少しブーストしてみる

  週 2: 「キーボードとボーカルの分離を良くする」
         → キーボードの 1-3kHz を少しカットしてみる

  週 3: 「配信のバランスを改善する」
         → 配信バスのミックスを別途調整する

  週 4: 「モニターの音量を下げても歌いやすくする」
         → モニター EQ の調整を試す
```

**1週間に1つ。欲張らない。** 毎週1つずつ改善すれば、1年で50個の改善ができます。

> **田中メモ:** 配信の改善は特に PDCA が回しやすい。録画をそのまま聴けばいいから。会場の音は録音しないと振り返れないけど、配信は毎回アーカイブが残る。これを活用しない手はない。

## 2. 録音して振り返る（10分）

### なぜ録音が必要か

```
  礼拝中のあなた:
  ┌──────────────────────────────────────┐
  │ フェーダー動かす → モニター調整 →    │
  │ 次の曲の準備 → 配信画面チェック →    │
  │ ワイヤレスのバッテリー確認 → ...     │
  │                                      │
  │ → 「全体としてどう聞こえていたか」    │
  │   を客観的に覚えていない！            │
  └──────────────────────────────────────┘

  だから録音が必要。
```

### 録音の方法

```
  方法 1: 配信のアーカイブを使う（最も簡単）
  ┌─────────────────────────────────────┐
  │ YouTube / Facebook の配信アーカイブ  │
  │ → そのまま聴ける                    │
  │ → 配信バスのミックスが聞ける        │
  │ ※ 会場の音とは違う点に注意         │
  └─────────────────────────────────────┘

  方法 2: ミキサーの録音機能を使う
  ┌─────────────────────────────────────┐
  │ デジタルミキサーの USB 録音          │
  │ → メインバスのステレオミックスを録音  │
  │ → 会場に近い音が録れる              │
  └─────────────────────────────────────┘

  方法 3: 会場にレコーダーを置く
  ┌─────────────────────────────────────┐
  │ 会衆席にポータブルレコーダーを設置    │
  │ → 会衆が実際に聴いている音を録れる   │
  │ → 部屋の残響も含めた「リアルな音」   │
  └─────────────────────────────────────┘
```

### 振り返りのチェックポイント

録音を聴くとき、以下の観点で聴きます。

```
  ┌──────────────────────────────────────────┐
  │ 1. ボーカル                               │
  │    □ 歌詞が聞き取れるか                    │
  │    □ 楽器に埋もれてないか                  │
  │    □ 各シンガーのバランスは取れているか    │
  │                                          │
  │ 2. 楽器                                   │
  │    □ 各楽器が聞き分けられるか              │
  │    □ 低域が濁ってないか                    │
  │    □ 全体のバランスは自然か                │
  │                                          │
  │ 3. 説教                                   │
  │    □ 声が明瞭に聞こえるか                  │
  │    □ 息遣いやポップノイズはないか          │
  │    □ 音量は適切か                          │
  │                                          │
  │ 4. 全体                                   │
  │    □ ノイズはないか                        │
  │    □ 音量の急な変化はないか                │
  │    □ 曲間の処理は自然か（フェードイン等）  │
  │                                          │
  │ 5. 配信（アーカイブで確認）                │
  │    □ 会場とのバランスの違いはないか        │
  │    □ 映像と音声のずれはないか              │
  │    □ ノイズや歪みはないか                  │
  └──────────────────────────────────────────┘
```

## 3. フィードバックを集める（5分）

自分の耳だけでは限界があります。色んな立場の人に聴きましょう。

```
  フィードバックの情報源:

  [会衆] ──→ 「今日の音はどうでしたか？」
              ※ 聞きすぎない。月1回程度で十分
              ※ 「聞こえにくかった場所はありますか？」が具体的

  [ミュージシャン] ──→ 「モニターは聞きやすかったですか？」
                       「自分の声/楽器は聞こえましたか？」
                       ※ 毎週リハ後に軽く確認

  [オンライン視聴者] ──→ 「音声はクリアでしたか？」
                         「音量は適切でしたか？」
                         ※ チャットやSNSでフィードバックを集める

  [配信アーカイブ] ──→ 自分で聴いて客観的にチェック
                       ※ これが最も確実
```

> **田中メモ:** 「音が良かった」と言われることは少ない。音響は「気づかれないのが最高の仕事」だから。でも「聞こえにくかった」「ノイズがあった」という指摘は宝物。改善のヒントを無料でもらえてるんだから、感謝して受け取ろう。

## 4. 部屋の音響を理解する（10分）

### 音は部屋で変わる

```
  同じミックスでも、部屋が違えば音が違う:

  【残響の多い部屋】（硬い壁・天井が高い・窓が多い）
  ┌──────────────────────┐
  │  ◎ → → →            │
  │     ↗ ↗ ↗（反射）    │
  │  ← ← ← ↙            │
  │     ↖ ↖ ↖（反射）    │
  │  → → → ↗             │
  │                      │
  │  音がグワングワン響く │
  │  言葉が聞き取りにくい │
  └──────────────────────┘

  【残響の少ない部屋】（カーペット・カーテン・吸音材）
  ┌──────────────────────┐
  │  ◎ → → → 会衆        │
  │     ↗ ░（吸音）       │
  │     ░（吸音）          │
  │                      │
  │  音がクリアに届く      │
  │  言葉が聞き取りやすい │
  └──────────────────────┘
```

### 定在波（Standing Waves）

```
  部屋の平行な壁の間で特定の周波数が増幅される:

  壁A                         壁B
  │←───────── 部屋の幅 ──────────→│
  │                               │
  │ ～～～～～～～～～～～～～～～ │ ← ある周波数が
  │～                           ～│   壁の間で共鳴
  │ ～～～～～～～～～～～～～～～ │
  │                               │
  │         ↑ここが              │
  │      音が大きい場所           │
  │      （腹: antinode）         │
```

**対策:** ミキシング位置が定在波の「腹」にあると、低域の判断を誤る。ミキサーの位置を少しずらしてみる。

### 簡単な音響改善

```
  ┌──────────────────────────────────────────┐
  │ 改善策            │ 効果       │ コスト  │
  ├──────────────────────────────────────────┤
  │ カーテンを閉める   │ 高域の反射減│ 0円    │
  │ カーペットを敷く   │ 床の反射減  │ 低     │
  │ 吸音パネル設置     │ 全帯域改善  │ 中     │
  │ 拡散パネル設置     │ 定在波改善  │ 中     │
  │ 天井に吸音材       │ 残響を減らす│ 中-高  │
  └──────────────────────────────────────────┘
```

### 部屋を「聴く」テスト

```
  簡易テストの手順:

  1. メインスピーカーからピンクノイズを流す
     （ミキサーのノイズジェネレーター or スマホアプリ）

  2. 会場を歩き回って聴く
     → 場所によって音の印象が違うはず

  3. 気づいたことをメモする:
     □ 後方の席は低域が多すぎる？
     □ 左右で音量差がある？
     □ 壁際で特定の周波数がブーンと響く？
     □ ステージ近くと後方で音質が違う？

  4. 問題がある場所を記録する
     → スピーカーの角度調整や EQ で補正
```

> **田中メモ:** 部屋の音響は一朝一夕には変えられない。でも「この部屋はこういう特性がある」と知っているだけで、ミックスの方針が変わる。残響が多い部屋なら EQ でクリアさを補う。デッドな部屋ならリバーブで少し空間を足す。部屋と対話するつもりでやろう。

## 6. 次世代を育てる（10分）

### なぜ引き継ぎが大切か

```
  ┌──────────────────────────────────────────┐
  │                                          │
  │  あなた1人しかPAできない状態 = 危険      │
  │                                          │
  │  ・あなたが休むと礼拝の音響が成り立たない │
  │  ・あなたが引っ越すと全てが止まる         │
  │  ・あなたが燃え尽きるリスクがある         │
  │                                          │
  │  → 最低2人、できれば3人のチームを目指す  │
  │                                          │
  └──────────────────────────────────────────┘
```

### 引き継ぎの3ステップ

```
  STEP 1: ドキュメントを整備する
  ┌──────────────────────────────────────┐
  │ □ 機材リスト（型番・設置場所）       │
  │ □ 接続図（何がどこに繋がっているか） │
  │ □ ミキサーのシーン設定の説明         │
  │ □ プリサービスチェックリスト         │
  │ □ 緊急対応カード                     │
  │ □ このカリキュラム（24レッスン分！） │
  └──────────────────────────────────────┘

  STEP 2: 段階的に任せる
  ┌──────────────────────────────────────┐
  │ 月 1: 横で見学してもらう             │
  │ 月 2: 説教パートだけ任せる           │
  │ 月 3: ワーシップも任せる（横でサポート）│
  │ 月 4: 1人でやってもらう（横で待機）   │
  │ 月 5: 完全に任せる                   │
  └──────────────────────────────────────┘

  STEP 3: 定期的にフォローする
  ┌──────────────────────────────────────┐
  │ □ 月1回、録音を一緒に聴いて振り返り  │
  │ □ 困ったことがないかヒアリング        │
  │ □ 新しい知識の共有                    │
  └──────────────────────────────────────┘
```

### ランシートを作る

毎週の礼拝用ランシート（進行表）を作っておくと、誰が PA をやっても最低限の進行ができます。

```
  ┌──────────────────────────────────────────────┐
  │         礼拝ランシート（例）                  │
  ├──────────────────────────────────────────────┤
  │ 時間   │ 内容           │ PA 操作            │
  ├──────────────────────────────────────────────┤
  │ 10:25  │ BGM 開始        │ iPhone ch ON       │
  │ 10:28  │ カウントダウン  │ PC ch ON           │
  │ 10:30  │ ワーシップ開始  │ iPhone ch OFF      │
  │        │                │ ボーカル・楽器 ON   │
  │        │                │ 配信・録画開始      │
  │ 11:00  │ ワーシップ終了  │ 楽器フェーダー下げ │
  │ 11:02  │ 説教開始        │ 説教マイク ON      │
  │ 11:40  │ 説教終了        │                    │
  │ 11:42  │ 応答の賛美      │ ボーカル・楽器 ON  │
  │ 11:50  │ アナウンス      │ MC マイク ON       │
  │ 11:55  │ 閉会            │ BGM → フェードアウト│
  │        │                │ 配信・録画停止     │
  └──────────────────────────────────────────────┘
```

> **田中メモ:** 配信の引き継ぎは特に大事。「配信ボタンを押し忘れた」は新人がやりがちなミス。ランシートに「配信開始」を明記して、チェックを入れる欄を作っておこう。画面の向こうの会衆を守るのは、次の世代のオペレーターの仕事でもある。

## 7. 来年の目標を立てる（5分）

PA の成長は段階的です。今の自分がどこにいて、次に何を目指すか。

```
  PA オペレーターの成長段階:

  Level 1: 操作ができる        ← Phase 1-2 で到達
  │  ミキサーの基本操作、ゲイン設定、フェーダー操作
  │
  Level 2: 音を作れる          ← Phase 3-4 で到達
  │  EQ、エフェクト、モニターミックス
  │
  Level 3: 問題を解決できる     ← Phase 5-6 で到達 ★今ここ

  │
  Level 4: 音を磨ける
  │  部屋の音響を理解し、繊細なミックスができる
  │  配信と会場のバランスを両立できる
  │
  Level 5: 人を育てられる
     次世代を指導し、チームを運営できる
```

### 来年の目標例


## 教会での応用

- **PDCA を習慣にする**: 毎週月曜に30分だけ録音を聴く。これだけで半年後には大きな差になる
- **配信品質の定期チェック**: 月に1回、スマホで自分の教会の配信を「初見の視聴者」として視聴する。音量・バランス・ノイズを客観的に確認

- **感謝を忘れない**: PA は見えない奉仕。でも、あなたの働きのおかげで会衆は礼拝に集中できている。オンラインの兄弟姉妹も、あなたのおかげで礼拝に参加できている
