## 今日のゴール
- 20以上のチャンネルをグループにまとめて一括操作できるようになる
- DCA グループとサブグループの違いを理解し、使い分けられる

Phase 3 まででゲイン設定、サウンドチェック、EQ の基本を学んだね。1チャンネルずつ丁寧に音を作れるようになった。

でも、礼拝本番を想像してみてほしい。目の前に 20 本以上のフェーダーが並んでる。賛美の最中に「ドラム全体をちょっと下げたい」と思ったら、キック・スネア・ハイハット・オーバーヘッドの 4 本を同時に動かす？ 無理だよね。

今日はその問題を解決する。**チャンネルをグループにまとめて、1本のフェーダーで一括コントロールする方法**を学ぶ。理由がわかれば忘れない --- これがルーティング設計だ。

## 1. なぜグループ分けが必要か（10分）

### フェーダー20本の現実

教会の入力チャンネルを全部並べてみよう:

```
ch1  ch2  ch3  ch4  ch5  ch6  ch7  ch8  ch9  ch10
[Vo1][Vo2][Vo3][Vo4][Sr1][Sr2][K-L][K-R][K-L][K-R]

ch11 ch12 ch13 ch14 ch15 ch16 ch17 ch18 ch19 ch20
[AG] [EG] [Bas][Kck][Snr][HH] [OH] [OH] [iPh][PC]
```

- Vo1-4 = ボーカル4本（ワイヤレス）
- Sr1-2 = 説教マイク2本
- K-L/R = キーボード2台（各ステレオ = 4ch）
- AG = アコギ、EG = エレキ、Bas = ベース
- Kck/Snr/HH/OH = ドラム（キック・スネア・ハイハット・オーバーヘッド）
- iPh = iPhone（BGM）、PC = カウントダウン

**20チャンネル。** 賛美中に「バンド全体をちょっと上げたい」と思ったら、14本のフェーダーを同時に動かす？ 不可能だ。

### グループの考え方

解決策はシンプル --- **似た役割のチャンネルを「グループ」にまとめる。**

```
┌─────────────────────────────────────────┐
│            礼拝のミックス全体            │
├────────┬────────┬────────┬──────────────┤
│ドラム  │ 楽器   │ボーカル│  その他      │
│グループ│グループ │グループ│ グループ     │
├────────┼────────┼────────┼──────────────┤
│Kck     │Keys1 L │Vo1    │Sr1           │
│Snr     │Keys1 R │Vo2    │Sr2           │
│HH      │Keys2 L │Vo3    │iPhone        │
│OH L    │Keys2 R │Vo4    │PC            │
│OH R    │AG      │       │              │
│        │EG      │       │              │
│        │Bass    │       │              │
└────────┴────────┴────────┴──────────────┘
```

グループにすれば、「ドラム」のフェーダー1本で 5ch 分を同時に上げ下げできる。

> **中村メモ:** 個々のチャンネルのバランスは事前に作っておく。グループフェーダーは「全体の音量」を動かすためのもの。ドラムの中でスネアだけ上げたい場合は、個別チャンネルを触る。

## 2. DCA グループとサブグループ（15分）

デジタルミキサーには、グループを作る方法が2つある。**DCA グループ**と**サブグループ（バス）**だ。

### DCA グループ（推奨）

DCA = Digital Controlled Amplifier（デジタル制御アンプ）。

**仕組み:** 複数チャンネルのフェーダーを「リモコン」で一括操作する。

```
ch1 [Vo1] ──┐
ch2 [Vo2] ──┼── DCA 1 [Vocals] ──→ そのまま Main L/R へ
ch3 [Vo3] ──┤
ch4 [Vo4] ──┘

※ 音声信号は各チャンネルから直接 Main へ流れる
※ DCA は「音量のリモコン」として機能するだけ
```

**特徴:**
- 音質が変わらない（信号経路に入らない）
- フェーダー操作が軽い（リモコンだから）
- ミュートも一括でできる
- **EQ やエフェクトは各チャンネルの設定がそのまま効く**

### サブグループ（バス）

**仕組み:** 複数チャンネルの音声信号を1つのバスにまとめてから Main へ送る。

```
ch1 [Vo1] ──┐
ch2 [Vo2] ──┼──→ Bus 1-2 [Vocals] ──→ Main L/R
ch3 [Vo3] ──┤     (ここで EQ や
ch4 [Vo4] ──┘      コンプをかけられる)

※ 音声信号がバスを「経由」して Main に行く
※ バス上で追加の処理ができる
```

**特徴:**
- グループ全体に EQ やコンプレッサーをかけられる
- 信号経路が1段増える（わずかに遅延が増える可能性）
- **ドラムバスにコンプをかける**のはよくある使い方

### どちらを使う？

| 用途 | 推奨 | 理由 |
|------|------|------|
| ボーカル全体の音量調整 | DCA | 音質を変えずに一括操作したい |
| ドラム全体の音量調整 | サブグループ | グループ全体にコンプをかけたい |
| 説教マイクの管理 | DCA | シンプルなミュート/音量操作だけ |
| 楽器全体の調整 | DCA | 個別のEQを活かしたまま操作したい |
| 再生音声の管理 | DCA | iPhone/PCの切替をシンプルに |

> **中村メモ:** 初心者のうちは**全部 DCA でOK**。サブグループはドラムバスコンプを使いたくなったときに導入すればいい。シンプルに始めて、必要に応じて複雑にする。これが鉄則。

## 3. 教会バンド向けルーティング設計（15分）

うちの教会の機材構成に合わせた、具体的なグループ設計を作ろう。

### チャンネルリストとグループ割り当て

```
┌──────┬────────────┬──────────┬───────────┐
│ ch   │ 入力       │ グループ │ DCA / Bus │
├──────┼────────────┼──────────┼───────────┤
│  1   │ Vocal 1    │ Vocals   │ DCA 1     │
│  2   │ Vocal 2    │ Vocals   │ DCA 1     │
│  3   │ Vocal 3    │ Vocals   │ DCA 1     │
│  4   │ Vocal 4    │ Vocals   │ DCA 1     │
├──────┼────────────┼──────────┼───────────┤
│  5   │ Sermon 1   │ Sermon   │ DCA 2     │
│  6   │ Sermon 2   │ Sermon   │ DCA 2     │
├──────┼────────────┼──────────┼───────────┤
│  7   │ Keys1 L    │ Keys     │ DCA 3     │
│  8   │ Keys1 R    │ Keys     │ DCA 3     │
│  9   │ Keys2 L    │ Keys     │ DCA 3     │
│ 10   │ Keys2 R    │ Keys     │ DCA 3     │
├──────┼────────────┼──────────┼───────────┤
│ 11   │ Acoustic G │ Guitars  │ DCA 4     │
│ 12   │ Electric G │ Guitars  │ DCA 4     │
├──────┼────────────┼──────────┼───────────┤
│ 13   │ Bass       │ Bass     │ DCA 5     │
├──────┼────────────┼──────────┼───────────┤
│ 14   │ Kick       │ Drums    │ DCA 5     │
│ 15   │ Snare      │ Drums    │ DCA 5     │
│ 16   │ Hi-Hat     │ Drums    │ DCA 5     │
│ 17   │ OH L       │ Drums    │ DCA 5     │
│ 18   │ OH R       │ Drums    │ DCA 5     │
├──────┼────────────┼──────────┼───────────┤
│ 19   │ iPhone BGM │ Playback │ DCA 6     │
│ 20   │ PC Count   │ Playback │ DCA 6     │
└──────┴────────────┴──────────┴───────────┘
```

### DCA 割り当て一覧

```
DCA 1: Vocals    [Vo1, Vo2, Vo3, Vo4]     ... 賛美ボーカル
DCA 2: Sermon    [Sr1, Sr2]               ... 説教マイク
DCA 3: Keys      [K1L, K1R, K2L, K2R]     ... キーボード
DCA 4: Guitars   [AG, EG]                 ... ギター
DCA 5: Band      [Bass, Kck, Snr, HH, OH] ... リズム隊
DCA 6: Playback  [iPhone, PC]             ... 再生音声
```

### 全体の信号フロー

```
  各チャンネル(ch1-20)
       │
       ├── チャンネル EQ / コンプ
       │
       ├──────────────────→ Main L/R ──→ メインスピーカー
       │                                  サブウーファー
       │
       ├── DCA 1-6 で音量を一括制御
       │
       ├── Aux 1-4 ──→ モニター（次回 L14 で学ぶ）
       │
       └── Aux 5-6 ──→ エフェクト（L15 で学ぶ）
```

> **中村メモ:** ベースとドラムを同じ DCA 5（Band）にまとめたのは、リズム隊は一緒に上げ下げすることが多いから。もし DCA の数に余裕があれば、ドラムとベースを分けてもいい。ミキサーの DCA 数を確認しよう。

## 4. ネーミングとカラーコード（10分）

デジタルミキサーの大きな利点 --- **チャンネルに名前と色をつけられる。**

### 命名ルール

```
短く、一目でわかる名前にする:

✗ 悪い例: "Keyboard Stage Right Yamaha"（長すぎ）
✗ 悪い例: "ch7"（何の入力かわからない）

○ 良い例: "Key1 L"（キーボード1の左）
○ 良い例: "Vo 田中"（ボーカルの田中さん）
```

### カラーコード（推奨）

```
┌────────────┬────────┬────────────────────┐
│ グループ   │ 色     │ 理由               │
├────────────┼────────┼────────────────────┤
│ Vocals     │ 赤     │ 最も重要、目立つ色 │
│ Sermon     │ 黄     │ 注意喚起の色       │
│ Keys       │ 青     │ 楽器グループと区別 │
│ Guitars    │ 緑     │                    │
│ Drums/Bass │ 紫     │ リズム隊           │
│ Playback   │ 白     │ 機械系は無彩色     │
└────────────┴────────┴────────────────────┘
```

ミキサーの画面イメージ:

```
[赤][赤][赤][赤][黄][黄][青][青][青][青][緑][緑][紫][紫][紫][紫][紫][紫][白][白]
Vo1 Vo2 Vo3 Vo4 Sr1 Sr2 K1L K1R K2L K2R AG  EG  Bas Kck Snr HH  OHL OHR iPh PC
```

**色がついていると、20本のフェーダーの中から目的のチャンネルを瞬時に見つけられる。** 礼拝中に「あのマイクどこだっけ？」と探す時間はない。

> **中村メモ:** カラーコードはチームで統一すること。誰が操作しても同じ色なら迷わない。設定したら写真を撮って共有しよう。

## 5. シーン / スナップショット管理（10分）

デジタルミキサーの最強機能 --- **現在の設定をまるごと保存して、ワンタッチで呼び出せる。**

### シーンとは

```
シーン = ミキサーの全設定のスナップショット（写真）

保存される情報:
├── 全チャンネルのフェーダー位置
├── 全チャンネルの EQ 設定
├── 全チャンネルのミュート状態
├── DCA のアサイン
├── Aux 送りのレベル
├── エフェクトの設定
└── チャンネル名・色
```

### 教会で使うシーン例

```
┌──────────┬────────────────────────────────┐
│ シーン名 │ 内容                           │
├──────────┼────────────────────────────────┤
│ 00_INIT  │ 初期状態（全フェーダー下げ）   │
│ 01_BAND  │ バンド賛美用（全ch使用）       │
│ 02_SERM  │ 説教用（説教mic + BGM のみ）   │
│ 03_ACOUS │ アコースティック賛美（少人数） │
│ 04_PLAY  │ 再生音声のみ（ビデオ上映等）   │
│ 99_OFF   │ 全ミュート（撤収用）           │
└──────────┴────────────────────────────────┘
```

### リコール（呼び出し）の注意

**全部を一気にリコールすると危険。** たとえば賛美中にシーンを呼び出したら、フェーダーがバッと動いて音がガクッと変わる。

**安全なリコール方法:**
1. **リコールセーフ（Recall Safe）** を設定する
2. 変えたくないチャンネルを「リコール対象外」にしておく
3. たとえば「説教シーン」を呼び出すとき、メインフェーダーはリコールセーフにして急な音量変化を防ぐ

```
リコールセーフの例:

シーン "02_SERM" を呼び出すとき:
  ├── リコールする: ch5-6(説教mic) のミュート解除、ch1-18 のミュート
  ├── リコールセーフ: Main L/R のフェーダー（急な音量変化を防ぐ）
  └── リコールセーフ: モニター送り（ステージの音を急に変えない）
```

> **中村メモ:** シーンは「礼拝前に準備しておくもの」。礼拝中にシーンを切り替えるのは慣れてから。最初のうちは手動でフェーダーとミュートを操作しよう。シーンは「万が一設定が飛んだときの保険」として使う。

## 教会での応用

### 礼拝前の準備（グループ設定）

1. **チャンネルの名前と色を設定する** --- これが最初の一歩
2. **DCA にチャンネルを割り当てる** --- ミキサーの DCA 設定画面で操作
3. **シーンを保存する** --- 設定が完了したら必ず保存

### 礼拝中の操作

グループがあれば、礼拝中の操作がシンプルになる:

```
賛美中:
  DCA 1 (Vocals)  → メインの音量調整
  DCA 5 (Band)    → バンド全体の音量調整
  個別 ch          → 特定の楽器の微調整

説教に移行:
  DCA 1 (Vocals)  → ミュート（または下げる）
  DCA 3-5         → ミュート（バンド全体）
  DCA 2 (Sermon)  → フェーダーアップ

再生音声:
  DCA 6 (Playback) → iPhone/PC の音量を一括管理
```

### よくある失敗

- **グループに入れ忘れたチャンネルがある** → サウンドチェック時に全チャンネルの DCA 割り当てを確認
- **色を統一していない** → チームで決めて、設定を共有する
- **シーンを保存し忘れた** → 設定を変えたら必ず上書き保存
