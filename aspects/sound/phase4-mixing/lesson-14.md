# Lesson 14: モニターミックス --- ステージ上の音を作る

> **Phase 4 / L14: ミュージシャンが演奏しやすい「聴こえ方」を Aux センドで作る**
> **担当:** 中村響（リードインストラクター）
> **所要時間:** 約60分
> **前提:** Phase 1-3 完了

---

## 今日のゴール
- モニターミックスが必要な理由を理解する
- Aux センドを使って、メインとは独立した複数のミックスを作れる
- ミュージシャンごとに「何を聴きたいか」を把握して対応できる

---

前回、20以上のチャンネルを DCA グループにまとめて効率的に操作する方法を学んだ。でもあれは全部「メインスピーカーから出る音」--- つまり会衆が聴く音の話だった。

今日はもうひとつの大事な仕事、**ステージ上のミュージシャンが聴く音**を作る。これが「モニターミックス」だ。

想像してみてほしい。ボーカルの人がステージに立っている。メインスピーカーは会衆に向いてる。ドラムの生音はうるさい。キーボードのアンプは遠い。**自分の声がほとんど聞こえない。** これで上手に歌えるかい？ 無理だよね。

ミュージシャンが自分の音を聴けるようにする --- これがモニターの役割だ。

---

## 1. モニターの基本概念（10分）

### メインミックスとモニターミックスの違い

```
          ┌─────────────────────────┐
          │      ミキサー            │
          │                         │
          │  ┌── Main L/R ──→ メインスピーカー（会衆向き）
          │  │
各ch ─────┤  ├── Aux 1 ────→ モニター1（ボーカル用）
          │  │
          │  ├── Aux 2 ────→ モニター2（ドラマー用）
          │  │
          │  ├── Aux 3 ────→ モニター3（キーボード用）
          │  │
          │  └── Aux 4 ────→ モニター4（ギタリスト用）
          │                         │
          └─────────────────────────┘
```

**ポイント:** Main と各 Aux は**完全に独立したミックス**。Main でボーカルを大きくしても、Aux 1 のボーカルは変わらない。それぞれ別の「レシピ」で音を混ぜる。

### Aux センド（auxiliary send = 補助送り）とは

各チャンネルには「Main フェーダー」の他に「Aux 送り」のノブがある。

```
チャンネル 1（Vocal 1）の中身:

  ┌─────────────────┐
  │  入力（マイク）  │
  │      ↓          │
  │    ゲイン       │
  │      ↓          │
  │     EQ          │
  │      ↓          │
  │  ┌───┬───┬───┐  │
  │  │Ax1│Ax2│Ax3│  │  ← 各 Aux へ送る量を個別に設定
  │  └─┬─┴─┬─┴─┬─┘  │
  │    │   │   │    │
  │    ↓   ↓   ↓    │
  │  [Aux1][Aux2][Aux3] → 各モニターへ
  │      ↓          │
  │  Main フェーダー │  ← メインスピーカーへの音量
  │      ↓          │
  │   Main L/R      │
  └─────────────────┘
```

つまり **1つの入力信号を、複数の出力先に、それぞれ違う音量で送れる。** これが Aux センドの仕組みだ。

> **中村メモ:** Aux は「補助」という意味だけど、モニターミックスにおいては主役。デジタルミキサーなら Aux 8本以上使えるものが多い。教会では 4〜6本あれば十分。

## 2. プリフェーダーとポストフェーダー（10分）

Aux センドには2つのモードがある。**これを間違えると大事故になる。**

### プリフェーダー（Pre-Fader）

```
  入力 → ゲイン → EQ → [Aux 送り] → モニターへ
                         ↓
                    Main フェーダー → メインスピーカーへ

  ※ Aux 送りは Main フェーダーの「前（Pre）」で分岐する
  ※ Main フェーダーを動かしても、Aux の音量は変わらない
```

### ポストフェーダー（Post-Fader）

```
  入力 → ゲイン → EQ → Main フェーダー → [Aux 送り] → エフェクトへ
                                          ↓
                                    メインスピーカーへ

  ※ Aux 送りは Main フェーダーの「後（Post）」で分岐する
  ※ Main フェーダーを動かすと、Aux の音量も連動して変わる
```

### なぜモニターはプリフェーダーか

| 状況 | プリフェーダー | ポストフェーダー |
|------|---------------|-----------------|
| Main でギターを下げた | モニターのギターは変わらない | モニターのギターも下がる |
| Main でボーカルを上げた | モニターのボーカルは変わらない | モニターのボーカルも上がる |

**モニターがポストフェーダーだったら？** PA担当が Main のフェーダーを動かすたびに、ステージ上の聴こえ方が変わる。ボーカリストは「急に聞こえなくなった！」とパニックになる。

**だからモニターは必ずプリフェーダー。** ステージ上の音は PA 操作に影響されない。

> **中村メモ:** 逆に、エフェクト（リバーブなど）への送りはポストフェーダーにする。Main で下げた楽器のリバーブだけ残り続けたらおかしいからね。エフェクトの詳細は次回 L15 で。

## 3. モニタースピーカーの配置とハウリング（10分）

### フロアモニター（ウェッジ）の配置

```
          ステージ（上から見た図）
  ┌───────────────────────────────┐
  │                               │
  │  [Drums]          [Keys2]    │
  │     ▲Mon2                    │
  │                   ▲Mon3      │
  │  [Bass]   [AG] [EG]         │
  │                   ▲Mon4      │
  │     [Vo2][Vo1][Vo3][Vo4]    │
  │           ▲Mon1              │
  │                               │
  └───────────────────────────────┘
         ▼            ▼
    [Main L]      [Main R]     ← 会衆席に向いている
```

- **Mon1:** ボーカル用（メインボーカルの足元）
- **Mon2:** ドラマー用（ドラムセットの横）
- **Mon3:** キーボード用
- **Mon4:** ギタリスト用

### ハウリング（フィードバック）のリスク

モニターは**マイクに向かって音を出す**という、本質的にハウリングが起きやすい配置。

```
  ハウリングのループ:

  マイク → ミキサー → モニター → マイクが拾う → ミキサー → モニター → ...
    ↑                    │
    └────────────────────┘  ← 無限ループ = 「キーン！」
```

**ハウリング対策:**

1. **モニターの音量を上げすぎない** --- 最小限の音量で
2. **モニターの向き** --- マイクの「裏側」（感度が低い方向）に向ける
3. **マイクとモニターの距離** --- なるべく離す
4. **モニター EQ** --- ハウリングしやすい周波数を少しカットする
5. **インイヤーモニター（IEM）** --- スピーカーを使わないので根本解決（後述）

> **中村メモ:** ハウリングが起きたら**まずモニターの音量を下げる**。それが最速の対処法。原因究明は後でいい。礼拝中は「止める」が最優先。

## 4. ミュージシャンごとのモニターニーズ（10分）

ここが一番大事。**ミュージシャンが何を聴きたいかは、人によって全く違う。**

### 典型的なモニターミックスの内容

```
┌───────────────┬──────────────────────────────────────┐
│ ポジション    │ 聴きたい音（優先順）                 │
├───────────────┼──────────────────────────────────────┤
│ メインボーカル│ ① 自分の声 ② キーボード ③ クリック │
│               │   （音程の基準が欲しい）             │
├───────────────┼──────────────────────────────────────┤
│ コーラス      │ ① 自分の声 ② メインボーカル         │
│               │   （ハモリの基準が欲しい）           │
├───────────────┼──────────────────────────────────────┤
│ ドラマー      │ ① クリック ② ベース ③ ボーカル     │
│               │   （テンポとグルーヴの基準）         │
├───────────────┼──────────────────────────────────────┤
│ ベーシスト    │ ① 自分の音 ② キック ③ ボーカル     │
│               │   （キックと合わせたい）             │
├───────────────┼──────────────────────────────────────┤
│ キーボード    │ ① 自分の音 ② ボーカル ③ ドラム     │
│               │   （コード進行とリズムの基準）       │
├───────────────┼──────────────────────────────────────┤
│ ギタリスト    │ ① 自分の音 ② ボーカル ③ ドラム     │
│               │   （自分のアンプ音が近いので少なめ） │
└───────────────┴──────────────────────────────────────┘
```

### 共通するパターン

全員に共通するのは **「自分の音が一番大きく聴きたい」** ということ。そして次に **「合わせる相手の音」** が欲しい。

> **中村メモ:** クリック（メトロノーム）はドラマーとメインボーカルだけに送る。会衆に聞こえたら大事故。クリック用の Aux は絶対に Main に流さないこと。

## 5. モニターミックスの作り方（10分）

### ステップバイステップ

**ステップ 1: まず全部ゼロにする**

```
Aux 1（ボーカルモニター）:
ch1 [Vo1] : 0
ch2 [Vo2] : 0
ch3 [Vo3] : 0
...
ch20[PC]  : 0

※ 全チャンネルの Aux 1 送りをゼロにしてスタート
```

**ステップ 2: 本人の音を入れる**

```
Aux 1（ボーカルモニター）:
ch1 [Vo1] : ████████░░ 80%  ← まず自分の声だけ
```

ボーカリストに「自分の声が聴こえるか」確認。ちょうどいい音量を見つける。

**ステップ 3: 基準楽器を足す**

```
Aux 1（ボーカルモニター）:
ch1 [Vo1] : ████████░░ 80%
ch7 [K1L] : ████░░░░░░ 40%  ← 音程の基準
ch8 [K1R] : ████░░░░░░ 40%
```

「キーボードも聴こえるか」確認。

**ステップ 4: 必要に応じて追加**

```
Aux 1（ボーカルモニター）:
ch1 [Vo1] : ████████░░ 80%  本人の声
ch7 [K1L] : ████░░░░░░ 40%  キーボード
ch8 [K1R] : ████░░░░░░ 40%  キーボード
ch14[Kck] : ███░░░░░░░ 30%  キック（リズムの基準）
```

**原則: 足すのではなく、必要最小限にする。** 何でもかんでもモニターに入れるとハウリングのリスクが上がる。

### 「More Me」シンドローム

ミュージシャンが「もっと自分の音を大きくして！」と何度も言ってくる現象。

```
「もっと声を上げて」→ 上げる → 他が聞こえない →
「キーボードも上げて」→ 上げる → 全体が大きい →
「もっと声を上げて」→ 上げる → ハウリング！

  ┌→ 自分の音を上げて ─→ 他が聞こえない ─┐
  │                                        │
  └── 他も上げて ←── 全体が大きくなる ←──┘
       ↓
  最終的にハウリング
```

**対処法:**
- **上げるのではなく、他を下げる。** 「ボーカルをもっと聴きたい」なら、ボーカルを上げるのではなく、他の楽器を下げる
- **ミュージシャンに説明する:** 「モニター全体の音量には限界があるので、不要な音を下げて、聴きたい音を目立たせます」

> **中村メモ:** ミュージシャンとのコミュニケーションは PA の仕事の半分。「上げて」と言われたとき、本当に上げるべきか、他を下げるべきか、判断できるようになろう。

## 6. フロアモニター vs インイヤーモニター（5分）

### 比較表

```
┌─────────────┬───────────────────┬───────────────────┐
│             │ フロアモニター    │ インイヤー (IEM)   │
│             │ （ウェッジ）      │                   │
├─────────────┼───────────────────┼───────────────────┤
│ 形状        │ 床置きスピーカー  │ イヤホン          │
│ ハウリング  │ リスクあり        │ ほぼゼロ          │
│ 個別制御    │ 全員同じ音になり  │ 完全に個別の      │
│             │ がち（出力数限定）│ ミックスが可能    │
│ ステージ音量│ 大きくなる        │ 静かになる        │
│ コスト      │ 比較的安い        │ 送信機＋受信機    │
│             │                   │ が人数分必要      │
│ 自由度      │ 場所が固定        │ どこでも聴ける    │
│ 注意点      │ 位置で聴こえ方が  │ 耳の疲労に注意    │
│             │ 変わる            │ 音量を上げすぎない│
└─────────────┴───────────────────┴───────────────────┘
```

### 教会での現実的な選択

- **まずはフロアモニターで始める** --- 機材が揃っているなら
- **ドラマーは IEM が理想的** --- ドラムの生音が大きいので、フロアモニターでは自分の音が聴こえにくい
- **ボーカルもIEMの恩恵が大きい** --- ハウリングのリスクが消える
- **予算に応じて段階的に導入** --- 全員一気に IEM にする必要はない

> **中村メモ:** IEM は「片耳だけ」で使う人がいるけど、あまりおすすめしない。片耳が外の音、片耳がIEMの音で、脳が混乱して音量をどんどん上げてしまう。使うなら両耳で、低めの音量で。

## 教会での応用

### サウンドチェック時のモニター手順

1. **全 Aux をゼロにしてスタート**
2. **一人ずつ確認する**（L11 のサウンドチェック手順と同じ順番）
3. 本人の音を入れる → 確認 → 基準楽器を足す → 確認
4. **全員揃ったらバンド全体でリハーサル**
5. リハーサル中に「聴こえにくいところ」をメモ → 調整

### 礼拝中のモニター操作

```
基本: モニターは触らない

  礼拝中にモニターをいじると:
  ├── ミュージシャンの聴こえ方が急に変わる
  ├── 演奏に影響する
  └── ハウリングのリスクが上がる

例外: 明らかにハウリングしそうなとき → すぐに下げる
```

### ミュージシャンとのコミュニケーションのコツ

- **礼拝前に聞く:** 「モニターで何を聴きたいですか？」
- **ハンドサイン（手信号）を決めておく:**

```
  ミュージシャン → PA担当

  ☝️ (上に向ける) = 「モニター上げて」
  👇 (下に向ける) = 「モニター下げて」
  👌 (OK)         = 「ちょうどいい」
  👂 (耳を指す)   = 「自分の声が聴こえない」
```

- **礼拝後に振り返る:** 「モニターの聴こえ方はどうでしたか？」

**Lesson 15: エフェクト実践 --- リバーブとコンプレッサー**

モニターミックスで「素の音」を届ける方法を学んだ。次回は音に「味付け」を加える --- リバーブ（残響）とコンプレッサー（音量の自動調整）。使いすぎると台無しだけど、適切に使えば礼拝の音がぐっと良くなる。
