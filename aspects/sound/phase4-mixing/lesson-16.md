# Lesson 16: 礼拝オペレーション --- 流れに沿ったミキシング

> **Phase 4 / L16: 礼拝の「流れ」を理解し、場面に応じた PA 操作を一通りこなす**
> **担当:** 中村響（リードインストラクター）
> **所要時間:** 約60分
> **前提:** Phase 1-3 完了

---

## 今日のゴール
- 礼拝の各場面で必要な PA 操作を一通り理解する
- BGM のフェードイン/アウト、賛美→説教の切り替えを実践できる
- キューシート（進行表）を作成し、礼拝全体を見通して操作できる

---

Phase 4 の最終回。ルーティング（L13）、モニター（L14）、エフェクト（L15）--- 道具は全部揃った。

今日はこれらを**礼拝の流れの中で実際に使う**。PA オペレーターにとって礼拝は「ライブショー」だ。でもコンサートと違うのは、**PA が目立ってはいけない**ということ。

音響がうまくいっている礼拝では、誰も PA のことを気にしない。歌が聴こえる、説教が聴こえる、ただそれだけ。でもその「ただそれだけ」を支えるために、裏で何十もの操作をしている。

**「見えないけど、不可欠」--- それが PA オペレーターだ。**

---

## 1. 礼拝の全体構成を把握する（10分）

### 典型的な礼拝の流れ

```
時間軸 →

[プリサービス] → [賛美] → [説教] → [応答] → [クロージング]
   BGM            バンド    説教mic   バンド     告知・送り出し
   カウントダウン  ボーカル  PC再生    ボーカル   BGM

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PA の仕事:
  音量  ░░▓▓▓▓▓▓▓▓▓▒▒▒▒▒▒▒▓▓▓▓▓▓░░░
  緊張度 ░░▓▓▓▓▓▓▓▓▓▒▒▒▒▒▒▒▓▓▓▓▓▓░░░
        低  高    高   中      高    低
```

### 場面ごとの使用チャンネル

```
┌──────────────┬──────┬──────┬──────┬──────┬──────┐
│              │Pre   │賛美  │説教  │応答  │Close │
├──────────────┼──────┼──────┼──────┼──────┼──────┤
│Vocal (ch1-4) │ ─    │ ON   │ ─    │ ON   │ ─    │
│Sermon (ch5-6)│ ─    │ ─    │ ON   │ ─    │ ON   │
│Keys (ch7-10) │ ─    │ ON   │ ─    │ ON   │ ─    │
│Guitars(ch11) │ ─    │ ON   │ ─    │ ON   │ ─    │
│Bass (ch13)   │ ─    │ ON   │ ─    │ ON   │ ─    │
│Drums(ch14-18)│ ─    │ ON   │ ─    │ △    │ ─    │
│iPhone (ch19) │ ON   │ ─    │ ─    │ ─    │ ON   │
│PC (ch20)     │ ON   │ ─    │ △    │ ─    │ ─    │
└──────────────┴──────┴──────┴──────┴──────┴──────┘
ON = 使用  ─ = ミュート  △ = 場合による
```

> **中村メモ:** この表を頭に入れておくだけで、次にどのチャンネルをON/OFFするか迷わなくなる。あとでキューシートにまとめるから、今は全体像を掴んでおこう。

## 2. プリサービス --- 礼拝が始まる前（8分）

### BGM（iPhone）の操作

```
礼拝開始30分前:

  iPhone ch19: ミュート解除 → フェーダーをゆっくり上げる
               （3〜5秒かけてフェードイン）

  目標音量: 会話の邪魔にならない程度（BGM だから）
           → メーター -20dB 〜 -15dB くらい

  ┌──────────────────────┐
  │  フェードインの手の動き  │
  │                        │
  │  ∞(下) → ゆっくり → -20dB で止める
  │                        │
  │  ※ 一気に上げない      │
  │  ※ 3〜5秒かけて       │
  └──────────────────────┘
```

**BGM の選曲ポイント:**
- 歌詞なし or インストゥルメンタルが無難
- テンポは落ち着いたもの（会話の邪魔にならない）
- **音量は iPhone 側とミキサー側の両方で調整**
- iPhone 側は 70-80% くらいに固定、ミキサーで微調整

### カウントダウン（PC）の操作

```
礼拝開始5分前:

  1. BGM をフェードダウン（-25dB くらいまで下げる）
  2. PC ch20: ミュート解除 → カウントダウン映像の音声を上げる
  3. PC 音声の音量を確認（事前にテスト済みのはず）
  4. カウントダウンが 0 になったら → PC をミュート
  5. BGM もフェードアウト（完全に下げ切る）
```

> **中村メモ:** BGM と PC の切り替えは「クロスフェード」ではなく「BGM を下げてから PC を上げる」。2つの音が同時に鳴ると混沌とする。切り替えはシンプルに。

## 3. 賛美セット --- バンドミキシングの本番（12分）

### 賛美の開始

```
カウントダウン終了後:

  1. DCA 6 (Playback) をミュート
  2. DCA 1 (Vocals) のミュート解除
  3. DCA 3-5 (Keys, Guitars, Band) のミュート解除
  4. リバーブのリターンフェーダーを確認
  5. 礼拝リーダーの「始めます」の合図を待つ

  ※ バンドが演奏を始める前にミュート解除しておく
  ※ フェーダーはサウンドチェック時の位置のまま
```

### 曲中のダイナミクス管理

賛美は「静かな部分」と「盛り上がる部分」がある。PA はこの波に合わせる。

```
曲の構成とPA操作:

  イントロ → Verse 1 → Chorus → Verse 2 → Chorus → Bridge → Chorus
  ───────────────────────────────────────────────────────────────
  音量:
  ░░░░▒▒▒▒▒▒▓▓▓▓▓▒▒▒▒▒▒▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓
  静か  普通   大きい  普通   大きい  もっと大きい
  ───────────────────────────────────────────────────────────────
  PA の操作:
  様子見  軽く    そのまま  少し    そのまま  全体を
         調整    見守る   下げる   見守る   少し持ち上げ
```

**重要な原則:**

1. **基本はフェーダーを動かさない。** サウンドチェックで作ったバランスを信じる
2. **動かすのは DCA（グループ）フェーダー。** 個別チャンネルはいじらない
3. **大きく動かさない。** 1〜2dB の微調整で十分
4. **バンドの音量に PA が合わせる。** PA が音楽を「作る」のではない

### 曲間の切り替え

```
曲 A 終了 → 曲 B 開始

  パターン1: 曲が繋がっている（メドレー）
  → 何もしない。フェーダーはそのまま

  パターン2: MCがある（礼拝リーダーが話す）
  → バンドが止まる → ボーカル DCA はそのまま
  → 楽器 DCA を少し下げる（完全ミュートはしない）
  → MC が終わったら楽器 DCA を戻す

  パターン3: 完全に止まる（祈りの時間など）
  → 全 DCA を下げる → 静寂
  → 再開時はリーダーの合図で DCA を上げる
```

### ワーシップリーダーの合図を読む

```
よくある合図（教会によって異なる）:

  ✋ 手を上げる     → 「盛り上げて」or「繰り返し」
  ✋ 手を下げる     → 「静かに」
  ✋ 手を振る       → 「続けて」
  👆 指で合図       → 「次の曲へ」
  🎤 マイクに口を近づける → 「話します」

  ※ 事前にリーダーと「合図」を確認しておくこと
```

### 自発的なワーシップ（即興の賛美）

教会の賛美では、曲リストにない即興の賛美が始まることがある。

```
即興賛美が始まったら:

  1. パニックにならない
  2. フェーダーはそのまま（いじらない）
  3. ボーカルが聴こえているか確認
  4. キーボードが伴奏を始めたら、そのバランスを維持
  5. 静かになったら全体を少し下げる
  6. リーダーが次のセクションに進む合図を待つ

  ※ 「何もしない」のが最善であることが多い
```

> **中村メモ:** 即興ワーシップは PA 初心者が一番焦る場面。でも実は「何もしない」が正解なことが多い。バランスはサウンドチェックで作ってあるから、フェーダーを触らなくても大丈夫。

## 4. 説教 --- 言葉の明瞭さが全て（10分）

### 賛美から説教への切り替え

```
賛美最後の曲 → 説教への移行

  タイムライン:
  ──────────────────────────────────────────
  [最後の曲] → [静かに終わる] → [牧師が壇上へ] → [説教開始]
  ──────────────────────────────────────────
  PA 操作:
  1. 曲が終わる
  2. DCA 1 (Vocals) をフェードダウン → ミュート
  3. DCA 3-5 (楽器) をフェードダウン → ミュート
  4. リバーブリターンを下げる（残響を切る）
  5. DCA 2 (Sermon) のミュート解除
  6. 説教マイク ch5 or ch6 のフェーダー確認
  7. 牧師が「おはようございます」→ 声が出ているか確認
```

**切り替えの順序が大事:**

```
  ✗ 悪い例: バンドをミュートしてから説教マイクを上げる
            → 数秒の「無音」が発生 → 会衆が不安になる

  ○ 良い例: 説教マイクを先に準備（ミュート解除）
            → バンドが自然に終わる → シームレスに説教へ
```

### 説教中の操作

```
説教中にやること:

  □ 使っていないチャンネルをミュートする
    → ノイズフロア（環境ノイズ）を下げるため
    → 特にドラムのマイクは開いたままだとノイズを拾う

  □ 説教マイクの音量を注視する
    → 牧師は感情が高ぶると声が大きくなる
    → コンプが効いているはず（L15 で設定済み）
    → それでも大きすぎたらフェーダーを -1〜2dB 下げる

  □ 説教マイクの EQ を確認
    → 低域（100Hz以下）をカット（ポップノイズ対策）
    → 中域（2-4kHz）を少し持ち上げ（言葉の明瞭さ）
    → 高域（8kHz以上）は控えめに（歯擦音 sibilance 対策）
```

### 説教中のビデオ / 音声再生

```
牧師: 「ここで短い映像を見てみましょう」

  1. PC ch20 のミュート解除
  2. 説教マイク ch5/6 はそのまま（ミュートしない）
     → 映像の後にすぐ話し始めるから
  3. PC のフェーダーを上げる（映像の音声）
  4. 映像が終わったら PC をフェードダウン → ミュート
  5. 牧師が話し始める → 問題なし（マイクは開いたまま）
```

> **中村メモ:** 説教中は「退屈」に感じるかもしれないけど、油断しない。牧師がマイクから離れたり、急に大きな声を出したり、突然映像を流したり。常にミキサーの前にいて、メーターを見守ること。

## 5. 応答 / 招きの時間（5分）

### 説教から応答への移行

```
説教終了 → 応答の賛美

  牧師: 「目を閉じて...」（祈りの時間）

  PA 操作:
  1. バンドが静かに演奏を始める準備をしている
  2. DCA 3 (Keys) のミュート解除（キーボードが先に入ることが多い）
  3. キーボードが静かに弾き始める → 音量を確認
  4. 徐々に他の楽器とボーカルのミュートを解除
  5. 牧師のマイクはまだ開けたまま（祈りの言葉があるかも）
  6. 賛美が本格的に始まったら説教マイクをフェードダウン
```

**応答の賛美は静かに始まる。** いきなりフルバンドで入ることは少ない。PA も「静かに始めて、徐々に上げる」。

```
音量の推移:

  [説教]  →  [祈り]  →  [応答賛美]  →  [盛り上がり]
  ▓▓▓▓▓▓    ░░░░░░    ▒▒▒▒▓▓▓▓    ▓▓▓▓▓▓▓▓▓▓
  説教mic    静寂      Keys+Vo     Full Band

  ※ 急な音量変化を避ける
  ※ 祈りの静寂を壊さない
```

> **中村メモ:** 応答の時間は礼拝で一番デリケートな瞬間。会衆が祈っている。ここでフェーダーをガチャガチャ動かしたら台無し。最小限の操作で、自然に音楽を入れていく。

## 6. クロージング --- 送り出しまで（5分）

### 告知と最後の曲

```
応答終了 → 告知 → 送り出し

  1. 賛美が終わる → バンドをフェードダウン → ミュート
  2. 説教マイク（またはボーカルマイク）で告知
  3. 最後の曲（あれば）→ バンドのミュート解除
  4. 最後の曲が終わる → フェードアウト
  5. BGM を静かにフェードイン
  6. 会衆が退席するまで BGM を流す
  7. 全員退席 → BGM フェードアウト
  8. 全チャンネルミュート → 電源を落とす手順へ
```

### ポストサービスのBGM

```
最後の曲終了:

  1. DCA 1-5 をフェードダウン → ミュート
  2. DCA 6 (Playback) のミュート解除
  3. iPhone ch19 をゆっくりフェードイン（5秒かけて）
  4. 会話の邪魔にならない音量で維持
  5. 片付け開始の合図があったら → フェードアウト → 全ミュート
```

## 7. キューシート（進行表）を作る（10分）

### キューシートとは

礼拝の流れと PA 操作を時系列でまとめた表。**これが PA オペレーターの「台本」。**

### キューシートの例

```
┌────┬──────────────┬─────────────────┬────────────────────────┐
│ #  │ 場面         │ 音源            │ PA 操作                │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 1  │ BGM開始      │ iPhone          │ ch19 フェードイン      │
│    │ (-30min)     │                 │ -20dB で維持           │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 2  │ カウントダウン│ PC              │ ch19↓ ch20↑           │
│    │ (-5min)      │                 │                        │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 3  │ カウント終了 │ -               │ ch20 ミュート          │
│    │              │                 │ DCA 1,3-5 ミュート解除 │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 4  │ 賛美1曲目   │ バンド+ボーカル │ リバーブON確認         │
│    │ 「曲名A」    │                 │ DCA でバランス微調整   │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 5  │ 賛美2曲目   │ バンド+ボーカル │ テンポ変化に注意       │
│    │ 「曲名B」    │                 │                        │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 6  │ 賛美3曲目   │ バンド+ボーカル │ バラードなら           │
│    │ 「曲名C」    │                 │ リバーブ長めに         │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 7  │ 説教開始     │ 説教マイク      │ DCA 1,3-5 ミュート     │
│    │              │                 │ DCA 2 ミュート解除     │
│    │              │                 │ リバーブリターン↓      │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 8  │ 説教中映像   │ PC              │ ch20 ミュート解除      │
│    │              │                 │ 映像後 ch20 ミュート   │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 9  │ 応答の賛美   │ Keys→バンド    │ DCA 3 ミュート解除     │
│    │              │                 │ 徐々に DCA 1,4-5 も   │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 10 │ 告知         │ 説教 or Vo マイク│ バンド→ミュート       │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 11 │ 送り出し BGM │ iPhone          │ ch19 フェードイン      │
│    │              │                 │ -20dB で維持           │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 12 │ 終了         │ -               │ 全ミュート → 電源OFF   │
└────┴──────────────┴─────────────────┴────────────────────────┘
```

### キューシートの作り方

1. **礼拝リーダーから曲順と流れを事前にもらう**
2. 各場面の音源（誰が / 何が鳴るか）を書き出す
3. 各場面の PA 操作を具体的に書く
4. **印刷してミキサーの横に置く**

> **中村メモ:** キューシートは毎週作る。5分で書ける。でもこの5分が礼拝中の安心感を生む。「次に何が来るかわかっている」--- これだけで落ち着いて操作できる。

## 8. PA オペレーターの心構え（5分）

### 「見えないけど不可欠」

```
┌─────────────────────────────────────────┐
│                                         │
│  良い PA = 誰も PA のことを気にしない    │
│  悪い PA = 全員が PA のことを気にする    │
│                                         │
│  ハウリング → 全員が振り返る            │
│  音が小さい → 「聞こえない」と言われる  │
│  音が大きい → 「うるさい」と言われる    │
│  ちょうどいい → 何も言われない          │
│                                         │
│  「何も言われない」が最高の仕事          │
│                                         │
└─────────────────────────────────────────┘
```

### PA オペレーターの3つの原則

**1. 準備が9割**
- サウンドチェックで全てのバランスを作り込む
- キューシートを用意する
- 本番は「微調整」だけで済むのが理想

**2. 耳で聴いて、目でも確認**
- メーターだけ見ていると本当の音がわからない
- メーターが正常でも、会場の後ろで聴こえが悪いことはある
- 可能なら礼拝前に会場内を歩いて確認する

**3. 焦らない**
- 問題が起きたらまず「下げる」（音量を下げれば被害は最小限）
- トラブルは必ず起きる。完璧な礼拝はない
- ミスをしても顔に出さない。会衆は気づいていないことが多い

> **中村メモ:** PA は「縁の下の力持ち」。でも、牧師やリーダーと同じくらい大事な役割。音がなければ礼拝は成り立たない。自分の仕事に誇りを持って、毎週少しずつ上手くなっていこう。

## 実践ドリル（10分）

### ドリル 1: キューシートを書いてみる

自分の教会の礼拝プログラムを思い出して、キューシートを作ってみよう。

```
┌────┬──────────────┬─────────────────┬────────────────────────┐
│ #  │ 場面         │ 音源            │ PA 操作                │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 1  │              │                 │                        │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 2  │              │                 │                        │
├────┼──────────────┼─────────────────┼────────────────────────┤
│ 3  │              │                 │                        │
└────┴──────────────┴─────────────────┴────────────────────────┘
```

### ドリル 2: 場面切り替えの順番

以下の場面切り替えで、操作の順番を答えてみよう:

**賛美 → 説教:**
1. _____ を先に準備する（ミュート解除）
2. バンドが _____ する
3. DCA _____ をミュートする

### 最初の数回でよくあるミス

| ミス | 対策 |

| BGM を切り忘れて賛美と重なる | キューシートに「BGM ミュート」を書く |

## 次回予告

**Phase 5: 配信・録音**

Phase 4 完了、おめでとう。これで教会の礼拝を一通り PA で回せるようになった。

Phase 5 からは担当が田中光に交代する。テーマは「配信・録音」。会場にいる会衆だけでなく、画面の向こうにいる人にも良い音を届ける方法を学ぶ。FOH ミックス（会場の音）と配信ミックスは別物 --- その違いと作り方を身につけていこう。

> **中村メモ:** ここまで L9 から L16 まで、8レッスン付き合ってくれてありがとう。最初は電源の順番もわからなかったのに、今は礼拝全体の PA オペレーションを理解している。あとは実践あるのみ。最初はうまくいかないこともあるけど、毎週やれば必ず上手くなる。理由がわかれば忘れない --- Phase 4 で学んだことを胸に、自信を持って PA 席に座ってほしい。
