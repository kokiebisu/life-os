## 1. 映像信号フローの基礎（15分）

### カメラから配信プラットフォームまで

```
【映像の基本的な流れ】

  カメラ1 ──┐
  カメラ2 ──┼──→ スイッチャー ──→ 配信PC ──→ YouTube等
  カメラ3 ──┘    (映像切替)       (エンコード)  (配信)

  各段階の役割:
  ┌──────────┐  映像を撮影する。礼拝堂に複数台設置
  │ カメラ    │  例: 正面全体、講壇アップ、会衆側
  └────┬─────┘
       │ HDMI / SDI ケーブル
       ▼
  ┌──────────┐  複数のカメラ映像を切り替える
  │スイッチャー│  「今はこのカメラの映像を使う」を選択
  └────┬─────┘  例: ATEM Mini, Roland V-8HD
       │ HDMI / USB
       ▼
  ┌──────────┐  映像+音声をエンコード（圧縮）して
  │ 配信 PC   │  インターネットに送出する
  └────┬─────┘  ソフト: OBS, Streamlabs, vMix 等
       │ インターネット
       ▼
  ┌──────────┐  視聴者が閲覧するプラットフォーム
  │YouTube 等 │  YouTube Live, Facebook Live 等
  └──────────┘
```

### 教会でよくある構成

```
【典型的な教会の配信構成】

  カメラ1(正面)    カメラ2(講壇)    カメラ3(横)
     │                │                │
     │ HDMI            │ HDMI           │ HDMI
     ▼                ▼                ▼
  ┌──────────────────────────────────────────┐
  │           ビデオスイッチャー              │
  │           (例: ATEM Mini Pro)             │
  │                                           │
  │   入力1    入力2    入力3    入力4        │
  │   [CAM1]  [CAM2]  [CAM3]  [PC映像]      │
  │                             ↑             │
  │                         カウントダウン     │
  │                         歌詞表示 等       │
  │                                           │
  │   PGM OUT ────→ 配信PC (OBS)             │
  └──────────────────────────────────────────┘
```

### HDMI と SDI

```
接続ケーブルの種類:

  ┌────────┬──────────┬──────────┬────────────────┐
  │ 規格    │ 最大距離  │ コスト    │ 用途            │
  ├────────┼──────────┼──────────┼────────────────┤
  │ HDMI   │ 10-15m   │ 安い     │ 短距離。         │
  │        │          │          │ 教会の多くは     │
  │        │          │          │ これで十分       │
  ├────────┼──────────┼──────────┼────────────────┤
  │ SDI    │ 100m+    │ やや高い │ 長距離。         │
  │        │          │          │ 大きな礼拝堂    │
  ├────────┼──────────┼──────────┼────────────────┤
  │ NDI    │ LAN経由  │ ソフト   │ ネットワーク     │
  │        │ 制限なし │ 次第     │ ベース。最新     │
  └────────┴──────────┴──────────┴────────────────┘

  教会の規模なら HDMI で十分なことが多い。
  15m 以上必要な場合は SDI か HDMI エクステンダーを使う。
```

## 2. 音声の映像信号への組み込み（10分）

### 音声はどこで映像に合流するか

```
【音声と映像の合流ポイント】

  パターン A: 配信 PC で合流（最も一般的）

    ミキサー ──→ 配信バス ──→ 配信 PC (OBS)  ──→ YouTube
                                  ↑
    スイッチャー ──→ 映像出力 ──┘

    OBS 内で音声と映像を合成して配信する


  パターン B: スイッチャーで合流

    ミキサー ──→ 配信バス ──→ スイッチャー ──→ 配信 PC ──→ YouTube
                                  ↑
    カメラ ────→ 映像 ──────┘

    スイッチャーが音声を埋め込んだ映像を出力する
    ※ ATEM Mini 等は音声入力端子あり
```

### パターン A（OBS で合流）の詳細

```
OBS の構成:

  ┌───────────────────────────────────────┐
  │               OBS Studio              │
  │                                       │
  │  映像ソース:                           │
  │  ┌─────────────────────────┐          │
  │  │ Video Capture Device    │          │
  │  │ (スイッチャーの出力)     │          │
  │  └─────────────────────────┘          │
  │                                       │
  │  音声ソース:                           │
  │  ┌─────────────────────────┐          │
  │  │ Audio Input Capture     │          │
  │  │ (ミキサーの配信バス)     │          │
  │  └─────────────────────────┘          │
  │                                       │
  │  出力 → YouTube Live / Facebook Live  │
  └───────────────────────────────────────┘

  ※ 映像と音声は別々の入力として OBS に入る
  ※ OBS が両方を合成してストリームにする
```

> **田中メモ:** パターン A が教会では最も一般的で、柔軟性も高いです。音声と映像を独立して調整できるので、同期のずれも OBS 内で補正しやすい。まずはこの構成を理解しましょう。

### 音声エンベデッド（パターン B）の注意点

```
スイッチャーの音声入力を使う場合:

  メリット:
  - 配信 PC の設定がシンプルになる
  - 映像と音声が1本のケーブルで PC に届く

  デメリット:
  - スイッチャーの音声入力の品質が限られることがある
  - 音声の細かい調整が OBS 側でやりにくい
  - トラブル時の原因切り分けが難しくなる

  → 教会では パターン A（OBS で合流）を推奨
```

## 3. 音声と映像の同期（10分）

### なぜずれるのか

音声と映像は、配信PC に届くまでの「処理時間」が異なります。

```
【遅延の発生ポイント】

  音声の経路:
  ミキサー ──→ 配信バス ──→ オーディオI/F ──→ PC
  遅延:        ほぼ0         1-5ms              1-5ms
  合計: 約 2-10ms（ほぼリアルタイム）

  映像の経路:
  カメラ ──→ HDMI ──→ スイッチャー ──→ キャプチャカード ──→ PC
  遅延:      1-3f     1-3f             1-5f                  処理
  合計: 約 50-150ms（音声より明らかに遅い）

  ※ f = フレーム。30fps の場合 1f ≒ 33ms

  結果:
  音声が映像より 40-140ms 早く届く
  → 「口が動く前に声が聞こえる」現象が起きる
```

### 同期の確認方法

```
手拍子テスト:

  1. カメラの前で手を叩く
  2. 配信（or OBS のプレビュー）で確認
  3. 手が合わさる映像と「パン」という音のタイミングを比較

  ┌─ 映像 ─┐   ┌─ 音声 ─┐
  │         │   │         │
  │  パチン │   │  パン   │
  │    ↓    │   │   ↓     │
  ─────●─────  ──●────────
       ↑            ↑
       映像の        音声の
       タイミング    タイミング

  この差が「同期ずれ」の量
```

### OBS での同期調整

```
OBS で音声の遅延を調整する方法:

  1. OBS を開く
  2. 「音声ミキサー」セクションで該当音声ソースの
     歯車アイコンをクリック
  3. 「プロパティ」→「同期オフセット (ミリ秒)」

  ┌─────────────────────────────────┐
  │ 音声ソース: ミキサー配信バス     │
  │                                 │
  │ 同期オフセット: [  80  ] ms     │
  │                                 │
  │ ※ 正の値 = 音声を遅らせる      │
  │ ※ 負の値 = 音声を早める        │
  └─────────────────────────────────┘

  調整手順:
  1. 手拍子テストでずれ量を測定（例: 約80ms音が早い）
  2. 同期オフセットに 80 と入力
  3. 再度手拍子テストで確認
  4. 微調整を繰り返す
```

> **田中メモ:** 同期調整は一度設定すれば、機材構成が変わらない限りそのまま使えます。ただし、カメラやスイッチャーを変更したら再調整が必要です。礼拝前のリハーサルで毎回確認する習慣をつけましょう。

## 4. 映像チームとの連携（10分）

### コミュニケーションプロトコル

```
【配信中の役割と連絡手段】

  音響担当              映像担当（スイッチャー操作）
  ┌──────────┐         ┌──────────┐
  │ ミキサー  │ ←─────→ │ スイッチャー │
  │ 配信音声  │  連絡    │ カメラ切替   │
  └──────────┘         └──────────┘
       │                      │
       └──────┬───────────────┘
              │
         配信PC担当（OBS操作）
         ┌──────────┐
         │ エンコード │
         │ 配信開始/停止│
         └──────────┘

  連絡手段:
  - インカム（トランシーバー）← 理想
  - 小声での直接会話 ← 近くに座る場合
  - テキストチャット ← 離れている場合
```

### 共有キューシート

```
キューシート例（礼拝の流れ）:

  時間    内容              カメラ        音響操作
  ──────────────────────────────────────────────────
  10:00   カウントダウン    PC映像        BGM フェードイン
  10:03   開会の挨拶       CAM2(講壇)    説教マイク ON
  10:05   賛美1曲目        CAM1(全体)    賛美ミックスへ
          間奏             CAM3(バンド)   楽器を上げる
          サビ             CAM1(全体)    ボーカル戻す
  10:10   賛美2曲目        CAM1→CAM3     同上
  10:15   献金の祈り       CAM2(講壇)    説教マイク ON
  10:20   説教             CAM2(講壇)    説教マイク中心
          聖書朗読         CAM2(講壇)    静かに
  10:50   応答賛美         CAM1(全体)    賛美ミックスへ
  10:55   祝祷・閉会       CAM2(講壇)    説教マイク ON
  11:00   後奏             PC映像        BGM フェードアウト
  ──────────────────────────────────────────────────

  ※ 映像と音響の「切り替えタイミング」を事前に共有する
```

> **田中メモ:** キューシートは礼拝のたびに更新してください。特に賛美曲が変わる場合や特別プログラムがある場合は重要です。「いつ何が起きるか」をチーム全員が知っていれば、スムーズな切り替えができます。

### タリーランプとカメラ切り替え

```
タリーランプの仕組み:

  カメラ1         カメラ2         カメラ3
  [  ]            [●赤]           [  ]
   ↑               ↑               ↑
  待機中          今ON AIR         待機中

  タリーランプ = カメラの上にある赤いランプ
  「今このカメラが配信に使われている」を示す

  音響担当にとっての意味:
  - タリーが点灯しているカメラの方向 = 映像の画角
  - 画角に映っている人のマイクが特に重要
  - 例: 講壇カメラ ON → 説教マイクの音質を特に注意
```

## 5. 配信プラットフォームの音声設定（5分）

### YouTube Live の音声設定

```
YouTube Live の推奨設定:

  OBS → 設定 → 出力 → 配信

  ┌─────────────────────────────────┐
  │ 映像ビットレート: 4500 kbps     │
  │                                 │
  │ 音声ビットレート: 192 kbps      │ ← ここが重要
  │ サンプルレート:   48 kHz        │
  │ チャンネル:       ステレオ      │
  │ エンコーダー:     AAC           │
  └─────────────────────────────────┘

  ※ 音声ビットレートは 128kbps 以上にする
  ※ ステレオで配信する（賛美の広がりが出る）
```

### モノラル vs ステレオ

```
配信をステレオにするか？モノラルにするか？

  ステレオ配信:
  ✅ 賛美の楽器が左右に広がる
  ✅ 音楽としての臨場感が出る
  ❌ スマホの片側スピーカーだと片ch しか聞こえない場合がある

  モノラル配信:
  ✅ どんなデバイスでも同じ音が聞こえる
  ✅ 安定性が高い
  ❌ 音楽の広がりがなくなる

  推奨: ステレオ配信
  ただし、配信バスのステレオ幅は控えめにする
  （極端に左右に振った音は、片側のみで聴く人に届かない）
```

## 7. 礼拝後の作業（5分）

### アーカイブのアップロード

```
礼拝後の配信チェックリスト:

  □ 配信を終了する（OBS で「配信終了」）
  □ 録音を停止し、ファイルを保存する
  □ YouTube のアーカイブを確認
     → 自動でアーカイブされているか
     → タイトル・説明文を設定
     → サムネイルを設定（可能なら）
  □ 録音ファイルを命名規則に従ってリネーム
  □ バックアップストレージにコピー
  □ ハイライト編集（必要な場合）
     → 賛美部分の切り出し
     → 説教部分の切り出し
     → SNS 用の短いクリップ作成
```

### ハイライト編集のワークフロー

```
配信アーカイブからコンテンツを作る:

  配信アーカイブ（2時間）
       │
       ├──→ 説教の切り出し（30-40分）
       │    → ポッドキャスト、YouTube 別動画
       │
       ├──→ 賛美ハイライト（3-5分）
       │    → SNS（Instagram, TikTok）
       │
       └──→ フルアーカイブ
            → YouTube にそのまま残す

  編集ツール:
  - 簡単な切り出し → OBS のリプレイバッファ
  - 本格編集 → DaVinci Resolve（無料）、iMovie
```

## 教会での応用

- **映像チームとの合同リハーサル:** 月に1回、音響チームと映像チームで合同リハーサルを行い、キューシートの確認とトラブル対応の練習をしましょう

## Phase 5 まとめ

Phase 5 では配信と録音の全体像を学びました。

```
L17: 配信の信号フロー
     → FOH と配信ミックスは別物。配信用 Aux バスを作る

L18: 録音の基礎
     → ステレオとマルチトラック。48kHz/24bit/WAV で録る

L19: 配信品質の管理
     → -14〜-16 LUFS。トラブル対応フローチャート

L20: 映像との連携
     → 音声と映像の合流。同期調整。チーム連携
```

音響の基礎から始まり、機材、セットアップ、ライブミキシング、そして配信・録音まで。あなたはもう教会の音響チームの一員として、自信を持って奉仕できるだけの知識を身につけました。

もちろん、知識だけでは十分ではありません。実践を重ねて、耳と手と判断力を鍛えていってください。そして忘れないでほしいのは、画面の向こうにも会衆がいるということ。あなたの奉仕は、会場にいる方だけでなく、遠く離れた場所で礼拝に参加している方にも届いています。
