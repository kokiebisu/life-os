> **所要時間:** 約60分
> **前提:** Phase 1-4 完了

## 今日のゴール
- ステレオミックス録音とマルチトラック録音の違いと使い分けを理解する
- 録音フォーマット（WAV/MP3、サンプルレート、ビット深度）の基礎を理解する
- 録音ワークフロー（準備→録音→保存→管理）を実践できるようになる

---

前回、配信ミックスの作り方を学びましたね。配信は「リアルタイムで届ける」技術でした。今日学ぶ録音は「未来の自分と他者のために残す」技術です。

画面の向こうにも会衆がいる。でも、それは今この瞬間だけではありません。来週、来月、あるいは何年も後に、録音を通じて礼拝のメッセージに触れる方がいるかもしれないのです。

録音はまた、あなた自身の成長のための最高のツールでもあります。ライブミキシングは一度きりですが、録音があれば何度でも振り返って改善点を見つけられます。

## 1. なぜ録音するのか（5分）

教会で録音する目的は大きく3つあります。

```
録音の3つの目的:

  1. アーカイブ     礼拝メッセージを記録として残す
     📁             → 欠席した方が後で聴ける
                    → 教会の歴史として保存

  2. コンテンツ     ポッドキャスト、SNS、ウェブサイト用の素材
     📱             → 説教の音声配信
                    → 賛美のハイライト動画

  3. スキル向上     自分のミキシングを振り返る
     📈             → 「あそこでボーカルが埋もれた」を客観的に確認
                    → 次回への改善点を見つける
```

> **田中メモ:** 特に3番目が重要です。ライブミキシングは本番中に「戻ってやり直し」ができません。でも録音があれば、後からゆっくり「ここはこうすべきだった」と分析できる。録音は最高の先生です。

## 2. 2つの録音アプローチ（15分）

### ステレオミックス録音

ミキサーの出力（2チャンネル: L/R）をそのまま録音する方法です。

```
【ステレオミックス録音の信号フロー】

  Ch1 ──┐
  Ch2 ──┤
  Ch3 ──┼──→ ミキサー内部 ──→ Main L/R ──→ 録音機器
  ...   │    ミキシング         (2ch)       (PCまたは
  Ch26 ─┘                                   レコーダー)

  録音されるもの:  ステレオ2トラック（最終ミックス）
  ファイルサイズ:  小さい
  後から調整:      できない（EQ やバランスは変更不可）
```

**メリット:**
- シンプル。接続するだけ
- ファイルサイズが小さい
- すぐにポッドキャスト等に使える

**デメリット:**
- 後からバランスを変更できない
- 「ボーカルをもう少し上げたい」ができない
- ミキシングのミスがそのまま残る

### マルチトラック録音

各チャンネルを個別に録音する方法です。

```
【マルチトラック録音の信号フロー】

  Ch1 ──┬──→ ミキサー ──→ Main L/R ──→ スピーカー
  Ch2 ──┤        │
  Ch3 ──┤        │
  ...   │        ▼
  Ch26 ─┘    USB / Dante / MADI
              │
              ▼
         PC (DAW ソフト)
         ┌─────────────┐
         │ Track 1: Ch1 │
         │ Track 2: Ch2 │
         │ Track 3: Ch3 │
         │ ...           │
         │ Track 26:Ch26 │
         └─────────────┘

  録音されるもの:  各チャンネル個別（26トラック）
  ファイルサイズ:  大きい（ステレオの13倍）
  後から調整:      自由自在
```

**メリット:**
- 後から自由にミキシングできる
- 「説教だけ抽出」「賛美だけ抽出」が可能
- ミキシングの練習素材になる

**デメリット:**
- ファイルサイズが大きい
- DAW ソフトの知識が必要
- 後処理に時間がかかる

### どちらを選ぶか？

```
判断フローチャート:

  録音の主目的は？
      │
      ├── 説教アーカイブ（音声配信）
      │   └──→ ステレオミックス録音で十分
      │
      ├── ミキシング技術の向上
      │   └──→ マルチトラック録音が理想
      │
      ├── 音楽コンテンツ制作
      │   └──→ マルチトラック録音が必須
      │
      └── 両方やりたい
          └──→ 両方同時に録音する（推奨）

  ※ 多くのデジタルミキサーは両方同時に可能
```

> **田中メモ:** 教会の状況が許すなら、両方同時に録音することをおすすめします。ステレオミックスはすぐにアーカイブとして公開し、マルチトラックは後から自分の練習用に使う。これが理想的なワークフローです。

├──────────────┬──────────────┬─────────────────────┤

## 4. 録音フォーマットの基礎知識（10分）

### WAV vs MP3

```
┌──────────────────────────────────────────────────┐
│              WAV（非圧縮）                         │
│                                                    │
│  音質:     最高（元データそのまま）                │
│  サイズ:   大きい                                  │
│  用途:     録音・編集・マスタリング                │
│  拡張子:   .wav                                    │
│                                                    │
│  → 録音時は必ず WAV を使う                        │
├──────────────────────────────────────────────────┤
│              MP3（圧縮）                           │
│                                                    │
│  音質:     良好（人間にはほぼ違いがわからない）    │
│  サイズ:   WAV の約 1/10                           │
│  用途:     配布・ポッドキャスト・アーカイブ        │
│  拡張子:   .mp3                                    │
│                                                    │
│  → 配布時に WAV から変換して使う                   │
└──────────────────────────────────────────────────┘
```

### サンプルレートとビット深度

```
サンプルレート（1秒間に何回音を記録するか）:

  44,100 Hz (44.1kHz)  ← CD 品質。教会録音にはこれで十分
  48,000 Hz (48kHz)    ← 映像制作の標準。配信と合わせるならこちら
  96,000 Hz (96kHz)    ← プロ音楽制作。教会では不要

  推奨: 48kHz（配信との互換性を考慮）


ビット深度（音の細かさ＝ダイナミックレンジ）:

  16bit  ← CD 品質。ダイナミックレンジ 96dB
  24bit  ← プロ標準。ダイナミックレンジ 144dB
  32bit float ← 最新。クリップしても後から救える

  推奨: 24bit（余裕のあるダイナミックレンジ）
```

> **田中メモ:** 迷ったら「48kHz / 24bit / WAV」で録音してください。これが現在の標準であり、どのDAWでも問題なく扱えます。

### ストレージ容量の目安

```
1時間の録音に必要な容量:

  ステレオ WAV (48kHz/24bit):
    48000 × 24 × 2ch × 3600秒 ÷ 8 ÷ 1024³
    ≒ 約 1 GB / 時間

  マルチトラック WAV (48kHz/24bit, 26ch):
    約 0.5GB × 26ch = 約 13 GB / 時間

  ──────────────────────────────────
  月4回の礼拝（各2時間）の場合:

  ステレオのみ:     約 8 GB / 月
  マルチトラック:    約 104 GB / 月
  両方:             約 112 GB / 月

  → 1TB の外付け HDD で約9ヶ月分
  ──────────────────────────────────
```

## 5. DAW の基礎知識（5分）

DAW（Digital Audio Workstation）は、PC上で録音・編集・ミキシングを行うソフトウェアです。

```
無料・低コストで使える DAW:

  ┌────────────────┬────────────┬──────────────────┐
  │ ソフト名        │ 価格       │ 特徴              │
  ├────────────────┼────────────┼──────────────────┤
  │ Audacity       │ 無料       │ シンプル。         │
  │                │            │ ステレオ編集向き   │
  ├────────────────┼────────────┼──────────────────┤
  │ GarageBand     │ 無料(Mac)  │ 直感的。           │
  │                │            │ マルチトラック対応  │
  ├────────────────┼────────────┼──────────────────┤
  │ REAPER         │ $60        │ 高機能。           │
  │                │ (個人利用) │ マルチトラック最適  │
  ├────────────────┼────────────┼──────────────────┤
  │ Cakewalk       │ 無料       │ フル機能。         │
  │ (by BandLab)   │            │ Windows のみ       │
  └────────────────┴────────────┴──────────────────┘

  教会での推奨:
  - ステレオ編集だけなら → Audacity
  - マルチトラック編集もするなら → REAPER
```

> **田中メモ:** DAW の深い使い方は今日のスコープ外です。まずは「録音ボタンを押す→停止→保存」ができれば十分。編集スキルは録音を続ける中で少しずつ身につけていきましょう。

## 6. 録音ワークフロー（10分）

### 礼拝前（準備）

```
録音準備チェックリスト:

  □ PC の電源を入れ、DAW を起動する
  □ ミキサーと PC の USB 接続を確認する
  □ DAW で入力デバイス（ミキサー）を選択する
  □ 録音トラックを作成する
     - ステレオ: 1ステレオトラック
     - マルチ:   各チャンネル分のモノトラック
  □ 録音フォーマットを確認する（48kHz / 24bit / WAV）
  □ 保存先フォルダを確認する（容量に余裕があるか）
  □ テスト録音する（10秒程度）
     → 再生して音が入っているか確認
  □ PC のスリープ設定をオフにする
  □ 不要なアプリを閉じる（CPU/メモリ確保）
```

### 礼拝中（録音）

```
録音タイムライン:

  礼拝開始5分前  ──→  録音スタート（REC ボタン）
       │                  │
       │              ※ 早めに開始することで
       │                冒頭が切れるのを防ぐ
       │
  礼拝開始      ──→  録音レベルをモニター
       │                  │
       │              ※ メーターが赤に振れていないか
       │                定期的に確認する
       │
  礼拝終了      ──→  すぐに停止しない！
       │                  │
       │              ※ 残響が消えるまで待つ
       │                （5〜10秒）
       │
  録音停止      ──→  ファイルを保存する
```

> **田中メモ:** 録音中にPCがフリーズする最悪の事態に備えて、DAW の「自動保存」機能をオンにしておきましょう。REAPER なら「Preferences → General → Auto-save」で設定できます。5分間隔がおすすめです。

### 礼拝後（保存・管理）

```
ファイル命名規則（推奨）:

  YYYY-MM-DD_礼拝種別_内容.wav

  例:
  2026-02-22_主日礼拝_ステレオミックス.wav
  2026-02-22_主日礼拝_マルチトラック/
    ├── Track01_Vocal1.wav
    ├── Track02_Vocal2.wav
    ├── Track03_Vocal3.wav
    ├── Track04_Vocal4.wav
    ├── Track05_Sermon1.wav
    ├── Track06_Sermon2.wav
    ├── Track07_KB_L.wav
    ├── Track08_KB_R.wav
    ├── ...
    └── Track26_PC.wav


フォルダ構造（推奨）:

  録音アーカイブ/
  ├── 2026/
  │   ├── 01-January/
  │   │   ├── 2026-01-04_主日礼拝_ステレオミックス.wav
  │   │   ├── 2026-01-04_主日礼拝_マルチトラック/
  │   │   ├── 2026-01-11_主日礼拝_ステレオミックス.wav
  │   │   └── ...
  │   ├── 02-February/
  │   └── ...
  └── バックアップ手順.txt
```

### バックアップの重要性

```
3-2-1 バックアップルール:

  3つのコピーを持つ     元データ + バックアップ2つ
  2種類のメディアに     内蔵HDD + 外付けHDD（or クラウド）
  1つは別の場所に       クラウドストレージ（Google Drive 等）

  最低限のバックアップ:
  ┌────────────┐     ┌──────────────┐
  │ PC 内蔵    │ ──→ │ 外付け HDD   │
  │ (録音先)   │     │ (毎週コピー) │
  └────────────┘     └──────────────┘
                            │
                            ▼
                     ┌──────────────┐
                     │ クラウド      │
                     │ (月1アップ)  │
                     └──────────────┘
```

## 実践ドリル（10分）

### ドリル 1: 録音テスト（5分）

以下の手順で実際に録音テストをしてみましょう（ミキサーがなくても PC のマイクで代用可能）。

1. Audacity（または手持ちの DAW）を起動する
2. 録音フォーマットを 48kHz / 24bit に設定する
3. 録音ボタンを押して、10秒間何か話す
4. 停止ボタンを押す
5. 再生して音が入っているか確認する
6. 「名前をつけて保存」で WAV 形式で保存する
7. ファイルサイズを確認する

  1回の録音容量:
    ステレオ: ___時間 × 1 GB = ___ GB

## 教会での応用

- **まずはステレオ録音から始める:** マルチトラックは準備が大変なので、最初はミキサーの Main Out をステレオで録音するだけでOK。それだけでも説教アーカイブとして大きな価値があります
- **説教ポッドキャスト:** ステレオ録音した説教部分を切り出して MP3 に変換すれば、すぐにポッドキャストとして配信できます
- **ミキシング振り返り会:** マルチトラック録音ができるようになったら、月に1回、録音を聴き返して「ここはこうすべきだった」を話し合う時間を設けてみましょう
- **ファイル管理担当を決める:** 録音ファイルの命名・整理・バックアップは地味ですが重要な仕事。専任の担当者を決めると続きやすいです
