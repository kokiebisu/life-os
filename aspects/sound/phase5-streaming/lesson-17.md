でも、今日からは「画面の向こうにも会衆がいる」ということを常に意識してほしいのです。配信を見ている方は、会場の空気感も、隣の人の賛美の声も聞こえません。あなたがミキサーから送り出す音だけが、その方の礼拝体験のすべてです。

これは怖いことではなく、素晴らしい奉仕の機会です。一緒に学んでいきましょう。

## 1. FOH ミックスと配信ミックスの根本的な違い（15分）

### 会場で起きていること

FOH（Front of House）ミックスでは、スピーカーから出る音だけが会衆に届くわけではありません。会場全体が「楽器」のように機能しています。

```
┌──────────────────────────────────────────────────┐
│                    会場空間                        │
│                                                    │
│   🥁 ドラム ─── 生音が直接会衆に届く              │
│   🎸 アンプ ─── アンプの音が会場に広がる          │
│   🎤 会衆の声 ─ 周囲の賛美が聞こえる              │
│                                                    │
│   🔊 スピーカー ─── FOH ミックスの音              │
│                                                    │
│   👥 会衆が聞く音 = スピーカー + 生音 + 反響       │
└──────────────────────────────────────────────────┘
```

つまり FOH エンジニアは、**会場音を含めたトータルバランス**を作っています。ドラムが生音で十分に聞こえるなら、FOH のドラムフェーダーは控えめでいい。

### 配信で起きていること

```
┌──────────────────────────────────────────────────┐
│               配信リスナーの環境                    │
│                                                    │
│   🎧 ヘッドホン or スマホスピーカー                │
│                                                    │
│   配信リスナーが聞く音 = ミキサーから送った音だけ   │
│                                                    │
│   会場の生音 ───── 聞こえない                      │
│   会場の反響 ───── 聞こえない                      │
│   周囲の賛美 ───── 聞こえない                      │
└──────────────────────────────────────────────────┘
```

> **田中メモ:** 配信リスナーにとって、ミキサーから送られる信号がすべてです。FOH では「ドラム生音があるからフェーダー低めでOK」でしたが、配信では「フェーダーを上げなければドラムは存在しない」のです。

### 具体的な違いの比較

```
                  FOH ミックス          配信ミックス
─────────────────────────────────────────────────────
ドラム         生音があるので控えめ     しっかり上げる
ボーカル       会場PAで十分             さらに前に出す
リバーブ       会場の自然残響あり       控えめに（後述）
ベース         サブウーファーで体感     ヘッドホン前提で調整
会衆の声       自然に聞こえる           環境マイクで拾う場合も
全体音量       SPL で管理              LUFS で管理（L19で詳述）
```

## 2. 配信用 Aux バスの設計（15分）

### なぜ FOH 出力を直接配信に送ってはいけないか

よくある失敗パターンを見てみましょう。

```
【NG パターン: FOH 出力を直接配信に送る】

  Ch1-26 ──→ Main L/R Bus ──→ メインスピーカー
                    │
                    └──→ そのまま配信 PC へ
                         ↑
                    これが問題！

  結果:
  - ドラムが小さい（FOH では生音があるから）
  - ボーカルが埋もれる（会場では近いから聞こえた）
  - リバーブが足りない or 多すぎる
  - 全体的に薄く、迫力がない配信になる
```

> **田中メモ:** 「FOH の Main Out をそのまま配信に使っている教会がとても多いのですが、これは配信リスナーに『半分だけの音』を届けているようなものです。会場にいる方は残りの半分を生音で補えますが、配信の方にはそれがないのです。」

### 正しいパターン: 配信用 Aux バス

```
【OK パターン: 配信専用バスを作る】

              ┌──→ Main L/R Bus ──→ メインスピーカー
              │    （FOH ミックス）
  Ch1  ──┬───┤
  Ch2  ──┤   └──→ Aux Bus 7/8 ──→ 配信 PC
  Ch3  ──┤        （配信ミックス）
  ...    │          ↑
  Ch26 ──┘     各チャンネルの
               送りレベルを
               個別に調整できる！

  ※ バス番号は一例。教会のミキサー構成に合わせて選ぶ
```

### 各チャンネルから配信バスへの送りレベル

FOH のフェーダーとは独立して、配信バスへの送り量を設定します。

```
チャンネル          FOH フェーダー    配信 Aux 送り     理由
─────────────────────────────────────────────────────────────
ボーカル×4          0 dB             +2〜+3 dB        配信では主役
説教マイク×2        0 dB             +3 dB            最重要。聞き取りやすく
Keyboard L/R        -3 dB            0 dB             配信でも存在感を
アコギ              -5 dB            -2 dB            少し上げる
エレキ              -5 dB            -3 dB            アンプ音がないので補う
ベース              -3 dB            0 dB             ヘッドホンで聞く前提
ドラム（OH+各パーツ）-10 dB          -3 dB            生音の代わりに上げる
iPhone（BGM）       -5 dB            -5 dB            同程度でOK
PC（カウントダウン） -5 dB            -5 dB            同程度でOK
```

> **田中メモ:** これはあくまで出発点です。実際のレベルは、ヘッドホンで配信ミックスを聴きながら調整してください。数字を覚えるより「ヘッドホンで聴いて自然か？」が判断基準です。

## 3. 配信ミックスのポイント（15分）

### ボーカルを前に出す

配信リスナーはイヤホンやスマホで聴いています。ボーカルが埋もれると歌詞が聞き取れず、礼拝への参加感が激減します。

```
FOH での理想バランス:        配信での理想バランス:

  ボーカル ████████░░        ボーカル ██████████
  楽器全体 █████████░        楽器全体 ███████░░░

  ※ FOH は楽器と同程度       ※ 配信はボーカルが明確に上
```

### リバーブの扱い

```
【FOH】                     【配信】
会場の自然残響 + エフェクト   エフェクトのリバーブだけ
→ 残響が十分                 → かけすぎるとボヤける

推奨:
- 配信バスには専用のリバーブを用意する
  （FOH のリバーブリターンとは別系統）
- リバーブタイム: 短め（1.0〜1.5秒）
- ウェット量: FOH の 50〜70% 程度
- プリディレイ: 20〜40ms（ボーカルの明瞭度を保つ）
```

### ドラムの扱い

Phase 4 で学んだように、FOH ではドラムの生音がかなり聞こえます。でも配信では違います。

```
配信でのドラムバランス:

  キック    ████████░░   しっかり入れる。低域の土台
  スネア    ███████░░░   存在感を出す
  ハイハット ████░░░░░░   控えめでOK
  OH (L/R)  ██████░░░░   シンバル類とドラム全体の空気感
  タム      █████░░░░░   フィル時に聞こえればOK

  ※ FOH より全体的にドラムの配信 Aux 送りを上げる
```

### ベースの注意点

```
配信リスナーの聴取環境:

  🎧 ヘッドホン   → 低域がダイレクトに届く。締まったベースが重要
  📱 スマホ       → 低域がほぼ再生できない。中域の倍音で存在感を出す
  💻 PCスピーカー → 低域は弱い。ヘッドホンとスマホの中間

  対策:
  - ベースは 80Hz 以下をハイパスで軽くカット
  - 800Hz〜1kHz あたりを少しブーストして「聞こえやすく」する
  - コンプレッサーでダイナミクスを揃える
```

## 4. 配信モニタリングの仕組み（10分）

### なぜモニタリングが必須か

FOH エンジニアは会場の音を自分の耳で聴いて調整します。配信エンジニアも同じです。**配信の音を聴かずに配信ミックスを作るのは、目をつぶって絵を描くようなもの**です。

### モニタリングの方法

```
【方法 1: ミキサーの配信バスをヘッドホンでモニター】

  配信 Aux Bus 7/8 ──→ ヘッドホンアウト
                         🎧 配信担当者

  メリット: レイテンシーなし、リアルタイムで確認
  デメリット: 実際の配信経路（エンコード後）は確認できない

【方法 2: 実際の配信をデバイスで確認】

  配信 PC ──→ YouTube/Facebook ──→ スマホ/タブレット
                                      📱 確認用

  メリット: 視聴者と同じ音質を確認できる
  デメリット: 5〜30秒の遅延がある
```

> **田中メモ:** 理想は両方やることです。配信担当者はヘッドホンでリアルタイムモニター。もう1人がスマホで実際の配信を確認。2人体制が難しければ、最低でもヘッドホンモニターは必ず行いましょう。

### ヘッドホン選びのポイント

```
配信モニター用ヘッドホンの条件:

  ✅ 密閉型（会場の音が入らない）
  ✅ フラットな周波数特性（音を正確に判断できる）
  ✅ 長時間装着しても疲れない

  ❌ 開放型（会場音が混ざって配信ミックスが判断できない）
  ❌ 重低音強調型（ベースの判断を誤る）
```

## 5. よくある失敗と対策（5分）

### 失敗 1: FOH を直接配信に送る（最頻出）

既に説明した通りですが、もう一度。

```
症状: 配信の音がスカスカ、ドラムが小さい、ボーカルが埋もれる
原因: FOH 出力 = 会場音込みの前提で作ったミックス
対策: 配信専用 Aux バスを作る
```

### 失敗 2: 配信ミックスを誰もモニターしていない

```
症状: 配信を見た人から「音が変だった」と事後報告
原因: 誰も配信の音をリアルタイムで聴いていなかった
対策: ヘッドホンモニター必須。担当者を決める
```

### 失敗 3: 配信にエコーが乗る

```
症状: 配信の音が二重に聞こえる、風呂場のように反響する
原因: 配信PCのスピーカーから出た音を会場マイクが拾っている
対策: 配信PCのスピーカーは必ずミュート。確認はヘッドホンで
```

### 失敗 4: 配信と会場で音量差がありすぎる

```
症状: 会場は大音量だが配信は小さい（またはその逆）
原因: 配信バスのマスターレベルが未調整
対策: 配信バスのマスターを -14〜-16 LUFS に合わせる（L19で詳述）
```

## 実践ドリル（10分）

### ドリル 1: 信号フロー図を描く（5分）

自分の教会のミキサーを想定して、以下を紙に描いてみてください。

1. 全チャンネル（ボーカル4本、説教2本、KB L/R、アコギ、エレキ、ベース、ドラム各パーツ、iPhone、PC）を左に書く
2. Main L/R バスと配信用 Aux バスを右に書く
3. 各チャンネルから両方のバスに線を引く
4. 配信バスの先に「配信PC」と「ヘッドホンモニター」を書く

### ドリル 2: 配信レベル設計（5分）

上で描いた信号フローの各チャンネルについて、FOH と配信で「同じ」「上げる」「下げる」を書き込んでください。

```
例:
  ボーカル1  FOH: 0dB  → 配信: 上げる (+2dB)
  キック     FOH: -10dB → 配信: 上げる (-3dB)
  ハイハット FOH: -8dB → 配信: 同じ (-8dB)
```

## 教会での応用

- **来週の礼拝で実験:** FOH の Main Out をヘッドホンで聴いてみてください。その音だけで礼拝に参加できるか？ おそらく「ドラムが小さい」「ボーカルが遠い」と感じるはずです。それが配信リスナーが聞いている音です
- **配信担当チームとの共有:** この信号フロー図を配信チームと共有し、「なぜ別ミックスが必要か」を説明してみましょう
- **既存の配信をチェック:** 教会の過去の配信アーカイブを聴いてみてください。FOH直送の問題が見つかるかもしれません
