## 1. 信号の流れ --- 全体像（15分）

### 基本の一本道

音の信号は、必ずこの順番で流れます。

```
  ┌────────┐   ┌────────┐   ┌──────────────┐   ┌────────┐   ┌────────┐   ┌──────────┐
  │  音源   │ → │ マイク  │ → │  ケーブル     │ → │ ミキサー│ → │ アンプ  │ → │ スピーカー │
  │(声/楽器)│   │(変換)   │   │(伝送)        │   │(調整)  │   │(増幅)  │   │ (出力)    │
  └────────┘   └────────┘   └──────────────┘   └────────┘   └────────┘   └──────────┘
   空気の振動→  電気信号に    ミキサーまで     音量・音質を   電気信号を    電気信号→
               変換          運ぶ            調整する       大きくする    空気の振動に
                                                                        戻す
```

左から右へ、一方通行。これが全ての基本です。

では、各ステージを詳しく見ていきましょう。

### ステージ 1: 音源（Sound Source）

教会にある音源を整理します：

```
  【マイクが必要な音源】          【直接ケーブルで繋ぐ音源（ライン入力）】
  ┌───────────────────┐         ┌───────────────────────┐
  │ ワイヤレスボーカルマイク ×4│         │ キーボード 1 (L/R)        │
  │ 説教マイク ×2           │         │ キーボード 2 (L/R)        │
  │ ドラム（複数マイク）     │         │ エレキギター（アンプ経由） │
  │ アコースティックギター   │         │ エレキベース（DI経由）     │
  └───────────────────┘         │ iPhone (BGM)              │
                                 │ PC (カウントダウン)        │
                                 └───────────────────────┘
```

> **藤井メモ:** 「マイクで拾う」のと「ケーブルで直接繋ぐ」のは、結果的にミキサーに電気信号が入るという点では同じ。でも信号の強さ（レベル）が違います。マイクの信号はとても弱い（マイクレベル）、ケーブル直の信号はそこそこ強い（ラインレベル）。ミキサーの入力にはそれぞれに合った受け口があります。

### ステージ 2: マイク / DI（変換）

**マイク** は空気の振動を電気信号に変換する装置です。

```
  空気の振動 → [マイク] → 微弱な電気信号
                           （マイクレベル: とても小さい）
```

**DI（ダイレクトボックス）** はエレキベースやキーボードなどの楽器信号を、ミキサーが受け取りやすい形に変換する装置です。

```
  楽器の出力 → [DI ボックス] → バランス信号
                                （ノイズに強い形に変換）
```

### ステージ 3: ケーブル（伝送）

マイクやDIからミキサーまで、ケーブルで信号を運びます。

```
  有線マイク:     XLR ケーブル ──────→ ミキサー
  ワイヤレスマイク: 電波 ──→ [受信機] ──→ ミキサー
  キーボード:      TRS/XLR ケーブル ──→ ミキサー
  iPhone:         3.5mm → ステレオDI ──→ ミキサー
```

教会のワイヤレスマイクは、実は「マイク → 送信機 → （電波）→ 受信機 → ケーブル → ミキサー」という経路です。電波の部分がケーブルの代わりをしています。

### ステージ 4: ミキサー（調整）--- ここが PA の心臓

ミキサーは信号の流れの中心です。次のセクションで詳しく見ます。

### ステージ 5: アンプ（増幅）

ミキサーから出てきた信号はまだスピーカーを動かすには弱いので、**パワーアンプ** で増幅します。

```
  ミキサー出力 → [パワーアンプ] → 大きな電気信号 → スピーカー
  （ラインレベル）  （増幅）      （スピーカーレベル）
```

> **藤井メモ:** 教会のスピーカーが「パワードスピーカー」（アンプ内蔵）の場合、アンプは外に見えません。スピーカーの中に入っています。外付けアンプがある場合は、ラック（機材棚）に入っていることが多いです。

### ステージ 6: スピーカー（出力）

スピーカーは電気信号を空気の振動に戻す装置です。マイクの逆。

```
  電気信号 → [スピーカー] → 空気の振動 → 会衆の耳
```

教会のスピーカーシステム：

```
  ┌────────────────┐
  │ メイン L/R     │ → 会衆席全体に向けて
  │（フルレンジ）   │    中〜高音を出す
  ├────────────────┤
  │ サブウーファー  │ → 低音専用
  │                │    ドラムやベースの重低音
  ├────────────────┤
  │ フロアモニター  │ → ステージ上の演奏者に向けて
  │                │    自分の音や他の楽器を返す
  └────────────────┘
```

## 2. ミキサーの中を詳しく見る（15分）

ミキサーは「信号が入ってきて、調整されて、出ていく」場所。中で何が起きているか見てみましょう。

### チャンネルストリップ --- 1本の信号が通る道

ミキサーに入ってきた各信号（ボーカル1、ボーカル2、キーボードL...）は、それぞれ **チャンネル** に割り当てられます。1つのチャンネルの中を **チャンネルストリップ** と呼びます。

```
  1つのチャンネルストリップ（上から下へ）:

  ┌─────────────────────────────┐
  │  入力（Input）               │ ← マイク/ライン信号が入る
  ├─────────────────────────────┤
  │  プリアンプ（Gain/Trim）     │ ← 信号の基本の大きさを決める
  │  [つまみで調整]              │    ※ここの設定がとても重要
  ├─────────────────────────────┤
  │  EQ（イコライザー）          │ ← 周波数ごとの音量を調整
  │  [High] [Mid] [Low]        │    Lesson 2 で学んだ帯域
  ├─────────────────────────────┤
  │  ダイナミクス                │ ← コンプレッサー/ゲートなど
  │  （Comp / Gate）            │    Lesson 3 のダイナミックレンジ管理
  ├─────────────────────────────┤
  │  Aux Send（補助出力）        │ ← モニターや配信用に分岐
  │  [Mon1] [Mon2] [Stream]    │
  ├─────────────────────────────┤
  │  Pan（パン）                 │ ← 左右の位置を決める
  │  [L ─────●───── R]         │
  ├─────────────────────────────┤
  │  フェーダー                  │ ← 最終的な音量を決める
  │  [  |||  ]                  │    Lesson 3 で学んだ dB
  ├─────────────────────────────┤
  │  → メインバス（L/R）へ      │ ← 全チャンネルがここに合流
  └─────────────────────────────┘
```

### バス --- 信号の合流地点

各チャンネルを通った信号は **バス（Bus）** に合流します。

```
  Ch1 (ボーカル1)  ──→─┐
  Ch2 (ボーカル2)  ──→─┤
  Ch3 (ボーカル3)  ──→─┤
  Ch4 (ボーカル4)  ──→─┤  ┌──────────────┐    ┌──────────┐
  Ch5 (説教マイク) ──→─┼→ │ メインバス    │ →  │ メイン出力 │ → スピーカー
  Ch6 (Keys L)     ──→─┤  │ （Main L/R） │    │ (L / R)  │
  Ch7 (Keys R)     ──→─┤  └──────────────┘    └──────────┘
  Ch8 (A.Guitar)   ──→─┤
  Ch9 (E.Guitar)   ──→─┤
  Ch10 (Bass)      ──→─┤
  Ch11 (Drums)     ──→─┤
  Ch12 (iPhone)    ──→─┤
  Ch13 (PC)        ──→─┘
```

メインバスの他にも、用途別のバスがあります：

```
  ┌──────────────┐
  │ メインバス    │ → メインスピーカー L/R（会衆向け）
  ├──────────────┤
  │ Aux 1       │ → フロアモニター 1（ボーカル用）
  ├──────────────┤
  │ Aux 2       │ → フロアモニター 2（楽器用）
  ├──────────────┤
  │ Aux 3       │ → サブウーファー（低音用）
  ├──────────────┤
  │ Aux 4       │ → 配信用ミックス（ストリーミング）
  └──────────────┘
```

> **藤井メモ:** Aux（オグジュアリー＝補助）バスが重要な理由は、「場所によって必要な音が違う」からです。会衆はバンド全体をバランスよく聞きたい。ボーカリストは自分の声をしっかり聞きたい。配信の視聴者はイヤホンで聞きやすいバランスが必要。それぞれ別のミックスを作れるのが、バスの仕組みです。

## 4. アナログ信号とデジタル信号（5分）

### アナログ信号

マイクが出す電気信号は **アナログ信号** です。音の波の形がそのまま電気の波になっています。

```
  音の波:     ╱╲  ╱╲  ╱╲
             ╱  ╲╱  ╲╱  ╲

  アナログ電気信号: ╱╲  ╱╲  ╱╲   ← そっくりそのまま
                   ╱  ╲╱  ╲╱  ╲
```

### デジタル信号

デジタルミキサーの中では、アナログ信号を **デジタル信号** に変換して処理しています。

```
  アナログ:  なめらかな波
             ╱╲  ╱╲  ╱╲
            ╱  ╲╱  ╲╱  ╲

       ↓ [AD 変換] ↓

  デジタル:  数字の列
             1, 5, 9, 12, 9, 5, 1, -5, -9, -12, -9, -5, ...
             （波の高さを数字にして記録）

       ↓ [ミキサー内で処理] ↓

       ↓ [DA 変換] ↓

  アナログ:  なめらかな波に戻す
             ╱╲  ╱╲  ╱╲
            ╱  ╲╱  ╲╱  ╲
```

- **AD変換**: アナログ → デジタル（ミキサーの入口）
- **DA変換**: デジタル → アナログ（ミキサーの出口）

> **藤井メモ:** デジタルミキサーの利点は、設定を保存できること。礼拝用、コンサート用、練習用......など場面ごとの設定を記憶して、ワンタッチで呼び出せます。アナログミキサーだと毎回ツマミを手で合わせ直す必要がありました。教会のデジタルミキサーには、きっと「日曜礼拝」用の設定が保存されているはずです。

## 5. ボーカルの声をたどる旅（10分）

最後に、実際のシナリオで信号の流れを追いかけてみましょう。賛美リーダーが歌い始めてから、会衆の耳と配信の視聴者に届くまでの全行程です。

```
  ╔══════════════════════════════════════════════════════════════╗
  ║         ボーカルの声が届くまでの旅（全行程）                    ║
  ╠══════════════════════════════════════════════════════════════╣
  ║                                                            ║
  ║  1. 声帯が振動する                                          ║
  ║     ↓                                                      ║
  ║  2. 口から空気の振動（音波）が出る                            ║
  ║     ↓                                                      ║
  ║  3. ワイヤレスマイクが音波をキャッチ                          ║
  ║     → 空気の振動が電気信号（アナログ）に変換される             ║
  ║     ↓                                                      ║
  ║  4. マイク内蔵の送信機が電気信号を電波に変換                   ║
  ║     → 電波が飛ぶ                                            ║
  ║     ↓                                                      ║
  ║  5. 受信機が電波をキャッチ                                    ║
  ║     → 電波が電気信号（アナログ）に戻される                     ║
  ║     ↓                                                      ║
  ║  6. XLR ケーブルでミキサーへ                                  ║
  ║     ↓                                                      ║
  ║  7. ミキサー入力 → AD 変換（アナログ→デジタル）               ║
  ║     ↓                                                      ║
  ║  8. プリアンプ（Gain）で基本レベルを調整                      ║
  ║     ↓                                                      ║
  ║  9. EQ で周波数バランスを調整                                 ║
  ║     （不要な低音をカット、声の芯を持ち上げるなど）             ║
  ║     ↓                                                      ║
  ║  10. コンプレッサーでダイナミクスを整える                     ║
  ║      （大きすぎる声を抑える）                                ║
  ║     ↓                                                      ║
  ║  11. フェーダーで他の楽器とのバランスを決める                 ║
  ║     ↓                                                      ║
  ║  ┌──── ここで信号が分岐する ────┐                            ║
  ║  ↓              ↓              ↓                           ║
  ║                                                            ║
  ║  【メイン】     【モニター】    【配信】                      ║
  ║  Main L/R      Aux 1-2        Aux (Stream)                ║
  ║  ↓              ↓              ↓                           ║
  ║  DA 変換        DA 変換        DA 変換                      ║
  ║  ↓              ↓              ↓                           ║
  ║  パワーアンプ    パワーアンプ    配信エンコーダー              ║
  ║  ↓              ↓              ↓                           ║
  ║  メインSPK     フロアモニター   インターネット               ║
  ║  ↓              ↓              ↓                           ║
  ║  空気の振動     空気の振動      視聴者のデバイス              ║
  ║  ↓              ↓              ↓                           ║
  ║  会衆の耳      演奏者の耳      視聴者の耳                    ║
  ║                                                            ║
  ╚══════════════════════════════════════════════════════════════╝
```

### 3つの「聞く人」がいる

PA操作者が常に意識すべきなのは、音を聞いている人が **3グループ** いるということ。

```
  ┌─────────────────────────────────────────────────┐
  │              PA操作者が面倒を見る3グループ         │
  ├──────────┬──────────┬──────────────────────────┤
  │ 会衆      │ 演奏者    │ 配信視聴者               │
  │          │          │                          │
  │ メインSPK │ モニター  │ 配信ミックス              │
  │ で聞く    │ で聞く    │ で聞く                   │
  │          │          │                          │
  │ 全体の    │ 自分の声  │ イヤホンで聞きやすい      │
  │ バランス  │ が欲しい  │ バランスが必要            │
  │ 重視     │          │                          │
  │          │          │ ダイナミックレンジ狭め     │
  └──────────┴──────────┴──────────────────────────┘
```

> **藤井メモ:** トラブルが起きたとき、信号の流れを「上流から下流へ」順番にチェックしていくと、必ず問題の場所が見つかります。「ボーカルマイク3の音が出ない」→ マイクの電源は入っている？ → 受信機のランプは光っている？ → ミキサーのチャンネルはミュートになっていない？ → フェーダーは上がっている？......と、上流から順に確認。これを **シグナルトレース** と呼びます。PA操作の基本スキルです。

書けたら、このレッスンの全体図と見比べてください。

### ドリル 2: 「音が出ない」トラブルシューティング

以下の状況で、信号の流れのどこに問題がありそうか考えてください：

1. **ボーカルマイク2だけ音が出ない。他のマイクは正常。**
   - ヒント: 他のマイクは出ているので、ミキサーから先は正常。マイク2固有の問題。

2. **全ての音が出ない。メインスピーカーから何も聞こえない。**
   - ヒント: 全チャンネルが同時に問題を起こすことはまずない。全チャンネルが合流する場所は？

3. **メインスピーカーは正常だが、配信に音が乗らない。**
   - ヒント: メインとストリームは分岐しているので、分岐後の問題。

## 教会での応用

今日学んだ信号の流れは、PA操作の全ての土台です：

1. **セットアップ手順の理由がわかる**: なぜ「マイクを繋いでからミキサーの電源を入れる」のか？ → 逆にすると、電源が入った状態で信号が突然入ってきて「バン！」とスピーカーが鳴る危険があるから

2. **トラブル対応ができる**: 音が出ないとき、信号の流れを上流から順にたどれば、どこで止まっているか見つけられる

3. **ミックスの考え方がわかる**: 各チャンネルの EQ やフェーダーは「その楽器の音をどう調整するか」、バスは「調整した音をどこに送るか」

4. **配信の音が会場と違う理由がわかる**: メインミックスと配信ミックスは分岐しているので、それぞれ別のバランスに調整できる（そして調整すべき）

> **藤井メモ:** Phase 1 はここまで。「音とは何か」「周波数」「デシベル」「信号の流れ」。この4つの基礎があれば、Phase 2 以降で学ぶ「EQの使い方」「コンプレッサー」「マイキング」「ミックスの作り方」が全部つながります。音には必ず理由がある。その理由を理解する力が、今日までのレッスンで身についたはずです。
