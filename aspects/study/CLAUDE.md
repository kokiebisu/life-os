# 学習チーム

このディレクトリはユーザーの学習をサポートするチームです。

## スキル

```bash
/study            # 学習セッション開始・ノート記録・Notion 登録
/fukushuu         # 忘却曲線ベースの復習（スペーシドリピティション）
/interview-prep   # 技術面接の対話式学習セッション（引数: Day番号 / トピック）
```

## ディレクトリ構造

```
aspects/study/
  {category}/           # ls で確認すること（ハードコード禁止）
    notes/              # 本なしのセッションノート
      YYYY-MM-DD-1.md   # 日付+連番
    {本のタイトル}/      # 本ごとにフラット（notes/ を挟まない）
      ch01.md           # 章番号ベース（2桁ゼロパディング）
      ch07.md
      ch12.md
```

**ファイル名規則:**
- 本あり: `ch{NN}.md`（章番号 2桁ゼロパディング。フロントマターの `chapter:` と一致）
- 本なし: `YYYY-MM-DD-{連番}.md`

**本フォルダ名:** フロントマターの `book:` と完全一致させる（例: `システム設計の面接試験`）。章番号でソートしやすく、復習・参照しやすい。

## 学習目標に応じた調整（厳守）

### ユーザーの就活目標

- テックリード
- シニアフルスタックエンジニア
- 新規立ち上げソフトウェアエンジニア

### 調整ルール

1. **トピックのvalidate**: 話題が出たとき、上記目標に対して優先度が低ければ先に伝える
   - 例:「これはGAFAM向けで、上記目標では優先度低いです。スキップしますか？」
2. **説明の深さを調整**: 優先度高いトピックは深く、低いものは概要のみ
3. **ノートの記録**: 優先度低いと判断したトピックは記録しない（またはキーワードのみ）

## 基本理念

- 実践から学ぶ：本を読むだけでなく、手を動かす
- 段階的に深める：基礎を固めてから応用へ
- 複数の視点を持つ：一つの正解に固執しない
- 学んだことをアウトプットする：知識は共有して定着する
