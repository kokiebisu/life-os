# Devotion - デボーション（聖書の対話型学び）

毎朝のデボーション。聖書の任意の書籍・章を一緒に読み、対話しながら深掘りし、記録を保存する。箴言に限らず、どの書籍でも扱う。

> **祈りは別スキル `/pray` で行う。** デボーションは聖書の学びに集中し、祈りは独立して呼ぶ。デボーション完了後に祈りたければユーザーが `/pray` を起動する。

## 引数

$ARGUMENTS — 章番号 or 日付（省略可。省略時は自動検出）

## Step 1: 準備

0. **セッション開始時刻を記録する（厳守・最初に実行）**
   ```bash
   TZ=Asia/Tokyo date +"%H:%M"
   ```
   この時刻を変数として覚えておく。後でNotionに登録するときの `--start` に使う。ユーザーには聞かない。

0.5. **仮置きエントリの早期検出**
   当日の Notion devotion DB に既存エントリがあるか確認する:
   ```bash
   bun run scripts/notion/notion-list.ts --date $(TZ=Asia/Tokyo date +%Y-%m-%d) --json
   ```
   - 「デボーション」を含むエントリが見つかった場合 → **ページ ID を記録しておく**（Step 3 で enrich に使う）。ユーザーに「既存エントリを見つけたので、完了後にそこに書き込みます」と軽く報告する
   - 見つからなかった場合 → 通常フロー（Step 3 で新規作成）

1. **前回の章を確認する**
   ```bash
   find aspects/devotions -name "*.md" | sort | tail -1
   ```
   最新ファイルを読んで前回の章番号を取得する（推測しない）

2. **テンプレートを生成する（厳守・スキップ禁止）**
   **必ず `devotion-init.ts` を実行してテンプレートファイルを生成すること。直接ファイルを作成してはならない。**
   ```bash
   # 引数なし → 自動で次の章・今日の日付
   bun run scripts/devotion-init.ts

   # 章を指定
   bun run scripts/devotion-init.ts --chapter $ARGUMENTS

   # 日付を指定
   bun run scripts/devotion-init.ts --date $ARGUMENTS
   ```
   - スクリプトが書籍・章番号の自動検出・重複チェックを行う。これを飛ばすと章の重複や番号ズレが起きる
   - 章数上限を超えた場合は自動で次の書籍の第1章に進む

3. **該当章を全文掲載する**
   - 章の全節を省略せず掲載する（聖書箇所の引用ルール: 勝手に抜粋しない）
   - **各節の冒頭に節番号を太字で付ける（例: `> **1** イエスは...`）。番号なしでの掲載は禁止**
   - ブロック引用（`>`）で記載し、末尾に書名・章・節を明記する

4. **`aspects/people/me.md` を全文読む（厳守）**
   - デボーションは家族・信仰遍歴・幼少期・価値観の話題に必ず触れる。**推測で書いて訂正されるより、最初に読む方が早く正確**
   - 該当セクションだけでなく全文を読む（背景は複数セクションに散らばる）
   - PK・牧師家庭・教会遍歴など、語る前提となる情報を頭に入れた状態で対話に入る

5. **ユーザーに声をかける**
   - 「XX書X章、一緒に読もう。気になった節はある？」
   - ユーザーが箇所を選ぶのを待つ

## Step 2: 対話（メインパート）

ユーザーが気になった節を起点に、対話で深掘りする。

### 対話の進め方
- ユーザーの発言を受け取り、一つのテーマを深く掘り下げる
- ヘブライ語の原語分析を交える（例: シェム、ヘン、タホール等）
- 聖書の他の箇所・人物を引用するとき、**必ず書名・章・節を添える**
- ユーザーの今の状況・感情と結びつけて語り合う
- `aspects/people/me.md` は Step 1-4 で全文読み済み。家族・信仰背景の前提を踏まえて語る

### 厳守事項
- **ユーザーの発言を先読みしない。** 言葉をそのまま受け取る
- **こちらから祈りで閉じようとしない。** ユーザーが「閉じよう」と言うまで対話を続ける
- **急いで次の節や結論に行かない。** 1時間たっぷり使う
- **まとめに急がない。** ユーザーが深く考えている時は待つ

## Step 3: 記録の保存（ユーザーが閉じたら）

ユーザーが「閉じよう」「まとめよう」等と言ったら:

1. **セッション全コンテンツを抽出してから記録する（厳守・スキップ禁止）**

   **書き始める前に、会話を遡って以下を箇条書きで列挙する:**
   - 話した節・聖句（節番号ごと）
   - 行ったヘブライ語・ギリシャ語の原語分析（単語・意味・気づき）
   - 引用した他の聖書箇所（書名・章・節）
   - ユーザーが語った個人的な接続・体験・感情
   - **ユーザーが投げかけた質問（全て列挙する。省略しない）**
   - 対話の中で浮かび上がったテーマや問い
   - 結論・洞察として到達した内容

   **抽出リストが揃ったら `aspects/devotions/YYYY-MM-DD.md` を完成させる:**
   - **深掘りセクションは「話したトピックごとに1つ」作る（1個固定ではない）。** 例: 2:4・2:10・2:24-25 を深掘りしたなら3セクション作る
   - 各深掘りセクションには、対話で実際に話した分析・原語・交差参照を全部書く（ハイライトのみにしない）
   - SOAP の Application には、ユーザーが語った個人的な接続を漏れなく書く
   - **`## 対話 Q&A` セクションを必ず作る（厳守）。** ユーザーが投げかけた質問を清書した形で `**Q:**` に書き、その場で答えた内容を `**A:**` に書く。質問数分だけ繰り返す。省略しない
   - **祈りは別スキル `/pray` の責務。** デボーション記録には書かない（Closing Prayer セクションは作らない）

   **書き終えたら網羅チェック（厳守）:**
   - 抽出リストの各項目がどこかのセクションに入っているか確認する
   - 漏れがあれば該当セクションに戻って補完してから次に進む

2. **フォーマット検証**
   ```bash
   bun run scripts/devotion-lint.ts $(TZ=Asia/Tokyo date +%Y-%m-%d).md
   ```

3. **Notion に同期する**
   - Notion **Devotion DB**（`NOTION_DEVOTION_DB`）の当日のエントリを探す:
     ```bash
     bun run scripts/notion/notion-list.ts --date $(TZ=Asia/Tokyo date +%Y-%m-%d) --json
     ```
   - エントリが見つかったら（仮置き enrich フロー）:
     - `notion-update-page` の `replace_content` でページ本文にローカル md と同じ内容を書き出す（省略・要約しない）
     - ステータスを「完了」に変更する
     - **Book（書籍名）と Chapter（数値）を設定する**（選択肢にない書籍は `notion-update-data-source` で追加してから設定する）
     - **icon を 🙏 に設定する**
     - **タイトルを「デボーション」に正規化する**（仮置き時に「朝デボ」「devotion」等の表記揺れがあり得るため）
     - **日付の start/end を実際のセッション時刻で更新する**（start: Step 1-0 で記録した時刻、end: `TZ=Asia/Tokyo date +"%H:%M"` で現在時刻を取得）。仮置き時刻はあくまで予定なので、実績で上書きする
   - エントリがなければ手動で登録する（**厳守**）:
     - **開始時刻**: Step 1-0 で記録した時刻を使う
     - **終了時刻**: `TZ=Asia/Tokyo date +"%H:%M"` で現在時刻を取得して使う
     - **ユーザーに時刻を聞かない（厳守）**
     ```bash
     bun run scripts/notion/notion-add.ts --db devotion --title "デボーション" --date YYYY-MM-DD --start HH:MM --end HH:MM
     ```
     その後 `notion-update-page` で以下を設定する:
     - Book（書籍名）と Chapter（数値）。Book の選択肢にない書籍は `notion-update-data-source` で追加してから設定する
     - icon: 🙏
     ※ Name は必ず「**デボーション**」（日本語）。分割登録（Part2 等）の場合も「**デボーション Part2**」のように「デボーション」をベースにする

4. **プロフィール更新（毎回実施）**
   - デボーション中にユーザーが語った個人情報・価値観・信仰の深まり・人生経験を振り返る
   - `profile/` の該当ファイルに追記すべき内容があれば更新する:
     - `profile/basic.md` — 事実・背景・信仰遍歴
     - `profile/personality.md` — 価値観・恋愛観・信仰の成長
   - 追記した内容をユーザーに1-2行で報告する

5. **完了報告**
   - 今日の章のタイトルと主な持ち帰りを1-2行で報告する
   - 祈りで閉じたい場合は `/pray` を案内する。**こちらから祈りに進まない**（`/pray` は独立スキル）

## 注意

- ROLE.md（`aspects/devotions/ROLE.md`）が正式なガイダンス。迷ったら参照する
- 聖書箇所を引用するときは**全文を書く**（参照だけで省略しない）
- 記録のフォーマットは ROLE.md のテンプレートに厳密に従う
- frontmatter title は単数形 `Devotion`（`Devotions` ではない）
- Key Verses は複数形
- SOAP は `## SOAP`（h2）
