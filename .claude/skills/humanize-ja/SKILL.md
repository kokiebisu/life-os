# humanize-ja — 外部発信文章を AI 臭から守る

ケンの就活用文章（履歴書・職務経歴書・面接準備・採用担当者向けメッセージ）を、AI 生成っぽさから守って本人の voice に寄せるスキル。

参照: [gonta223/humanizer-ja](https://github.com/gonta223/humanizer-ja)（汎用 humanizer の 20 パターン）。これをベースに、就活ドメイン特化のパターンとケン本人の voice 参照を追加した個人版。

## 起動条件

- `aspects/job/` 配下の md ファイルを **新規生成・大きく書き直す** とき → Claude が自動 invoke
- `/humanize <file>` で **既存ファイルを後から humanize** したいとき
- 採用担当者・面接官・転職エージェントなど **外部に出す日本語文章** を書くとき
- **PostToolUse hook 検知**: `aspects/job/` 配下を Edit/Write した直後、`scripts/humanize-check.sh` が AI 臭パターンを grep し、ヒットがあれば Claude に通知（ユーザー手動編集にも反応）

## ファイル構成

```
skills/humanize-ja/
├── SKILL.md         # このファイル（起動条件 + ワークフロー）
├── patterns.md      # NG パターン詳細
├── voice.md         # ケンの voice 4 特徴 + 参照サンプル + Before/After 例
└── checklist.md     # 出力前の最終確認 7 項目
```

## ワークフロー

### Mode 1: pre-write（新規生成時、自動起動）

`aspects/job/` 配下を新規生成・大きく書き直すとき:

1. **`voice.md` を読む** — ケンの voice 4 特徴を頭に入れる（モジュール化 / 定量化・固有名詞 / 逆算型 / 丁寧さとドライさの両立）
2. **`patterns.md` を読む** — NG リスト（humanizer-ja 流用 + 就活特化）を頭に入れる
3. **書く** — 4 特徴を体現し、NG パターンを避ける
4. **出力前に `checklist.md` の 7 項目をなぞる** — 漏れがあれば書き直す

### Mode 2: post-check（既存ファイル humanize、`/humanize <file>`）

1. **ファイル読み込み** — 対象 md を読む
2. **NG パターン検出** — `patterns.md` の表に基づき、行単位で AI 臭フレーズを抽出
3. **「削る + 埋める」のペアで提案** — 各ヒットに対し以下の形式で diff を提示:
   - **Before**: 該当行
   - **削った理由**: どのパターンに該当するか
   - **After**: 削った AI 臭フレーズ + そこに埋める具体事実
4. **ユーザー承認後に Edit 適用** — 一括承認 or 1 つずつ承認を選べるようにする
5. **適用後 `checklist.md` を再走** — 通過確認

## 重要な原則

### 主目標は「voice 4 特徴の体現」、AI パターン回避は副次目標

`voice.md` の 4 特徴（モジュール化 / 定量化・固有名詞 / 逆算型 / 丁寧 + ドライ）を体現することが第一。AI パターン回避はそのための手段。`patterns.md` のチェックリストに振り回されて、ケンらしさが失われたら本末転倒。

### 「削る」だけで終わらない、「埋める」までセット

AI 臭フレーズを削るだけだと、履歴書としての規模感・難易度が消える。**削った場所には必ず具体事実を埋める**（数字・固有名詞・具体的な状況描写）。`voice.md` の Before/After 例は 3 列構成（Before / 削った理由 / After + 具体事実）でこれを徹底する。

### カジュアル方向への振れ幅は 7 割で抑える

履歴書・職務経歴書は媒体特性として「初対面・5 分・流し読み」。「思ってます」「感じてます」のような口語動詞は使わない。ですます調は崩さない。詳細は `voice.md` D セクション参照。

### 数字は実測 or 根拠あり

After 列で具体数字を埋めるとき、面接で深掘りされて答えられない数字は使わない。確認できない場合は数字を削るか、別の角度の具体事実で埋める。

## 媒体別の注意

| 媒体 | カジュアル度 | 一人称 | 口語動詞 |
|---|---|---|---|
| 職務経歴書（Yagish 等） | 低（ですます調統一） | 「自分」も最小化 | NG |
| カバーレター | 中（ですます調 + 一部体言止め OK） | 「私」を使ってよい | 「考えています」程度なら可 |
| 採用担当者へのメール | 高（丁寧、ただし配慮優先） | 「私」 | NG |
| 面接対策ノート（自分用） | カジュアル可（自分用なので） | 自由 | 可 |

## 詳細リファレンス

- **NG パターン詳細** → `patterns.md`
- **voice 4 特徴 + Before/After 例** → `voice.md`
- **最終チェックリスト** → `checklist.md`
