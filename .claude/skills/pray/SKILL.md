# Pray - 祈り（Closing Prayer）

`aspects/church/prayer-requests.md` の Active リスト全員のために祈る。デボーションとは独立して呼べる。デボーション直後でも、別のタイミングでも、単独で実行できる。

## 引数

なし

## Step 1: Active リストを読む

```bash
cat aspects/church/prayer-requests.md
```

Active セクションのカテゴリ別の人名・祈りのテーマを把握する。

## Step 2: 変化を確認する（厳守・スキップ禁止）

祈り文を書く**前に**、Active リスト全員の名前と祈りのテーマをユーザーに表示する。その上で次のように聞く:

> 「変化はある？ 新しく加えたい人、状況が変わった人、Answered になった人いる？」

ユーザーの回答を待つ。確認なしに祈り文を書き始めない。

- 変化があれば `aspects/church/prayer-requests.md` を編集してから祈りに進む
- 新規追加・Answered 化があれば該当 `aspects/people/<name>.md` も更新する
- 変化なしと言われたらそのまま Step 3 へ

## Step 3: 祈り文を生成する

Active リスト全員を含む祈り文を生成し、画面に表示する（**ファイル保存はしない**）。

### フォーマット（厳守）

```markdown
> 父よ、今この時間にあなたの前に出ます。
> <ケンの今の状況・心境への短い応答 — 2〜5行、blockquote で改行ごとに `> ` を付ける>

そして今、Active な祈りリストの一人ひとりをあなたの前に持ち運びます。

### <カテゴリ名（Pastor / Staff、Fuse（Yokohama）など）>
- **<人名>** — <その人への祈りの要点（短く1行）>
- **<人名>** — <祈りの要点>

### <次のカテゴリ>
- **<人名>** — <祈りの要点>

### ケン自身
- <祈りの要点>
- <祈りの要点>

---

イエス様の御名によってお祈りします。アーメン。
```

### ルール

- 冒頭は `> ` blockquote。改行ごとに `> ` を付け、5行程度に収める
- カテゴリは `### ` h3 ヘッダー（インライン太字 `**...**` にしない）
- 各人は独立した bullet。1人1行が原則。1行に複数人を詰め込まない
  - 例外: Tidi 先生 / Fumie 姉妹のように Active リストでまとめて記載されている場合のみ `/` 区切りで 1 bullet にしてよい
- 人名は `**...**` で太字
- 締めの「イエス様の御名によって…」の前に `---` 区切りを入れる
- 祈りは**一人称（ケン視点）で書く。** 「この人を」ではなく「<人名>を覚えます」「<人名>に〜してくださいますように」

## Step 4: 完了

祈り文を出力したら終了。**ファイル保存・Notion 同期はしない**（祈りは画面に出して読む、その瞬間の祈りとして扱う）。

過去の祈り文を残したい場合は、ユーザーが自分で aspects/devotions/ など適切な場所に貼り付ける。スキル側では自動保存しない。

## 注意

- **こちらから祈りで閉じようとしない。** ユーザーが「祈ろう」と言ってからこのスキルを起動する
- **Active リストの全員を必ず含める。** 一人でも飛ばさない
- **テーマが `—`（未設定）の人も名前は呼ぶ。** 「<人名>を覚えます」など簡潔に
- prayer-requests.md の編集が必要になったときは `/from-notion` や手動編集で対応する。`/pray` は読むだけ（変化を反映する場合のみ書き込む）
