# discover-supply-chain — バリューチェーン起点の発掘

## いつ使う

- 「キオクシア型の一発を当てたい」「川上の部品・素材・装置系を探して」
- 「ニュースで騒がれる前の会社を見つけたい」
- `/discover-growth`（ニュース起点）では出てこない「地味な B2B 必需品企業」を探すとき

## ニュース起点との違い

| | /discover-growth | /discover-supply-chain |
|--|--|--|
| 起点 | ニュース見出し | メタトレンドのバリューチェーン構造 |
| 対象 | 話題になった成長株 | まだ注目されていない川上企業 |
| セクター | AI/Software 中心 | ハードウェア・素材・装置・エネルギー |
| 時価総額 | $1B-$20B | $500M-$30B（より小型も含む） |
| 発見タイミング | ニュースになってから | ニュースになる前 |

## 分析するメタトレンド（5 つ）

1. **AI データセンターインフラ** — GPU・HBM メモリ・電力・冷却・光配線
2. **ヒューマノイドロボティクス** — センサー・アクチュエーター・エッジ AI チップ
3. **宇宙経済インフラ** — 打ち上げ部品・衛星コンポーネント・地上局
4. **次世代エネルギーグリッド** — SMR 部品・グリッド近代化・電力半導体
5. **半導体製造装置・材料** — EUV 周辺・検査・特殊ガス・フォトレジスト

## 実行

```bash
# dry-run（JSON 出力なし）
bun run scripts/investment/discover-supply-chain.ts --dry-run

# 本番（candidates/ に JSON 出力）
bun run scripts/investment/discover-supply-chain.ts

# 候補数を指定
bun run scripts/investment/discover-supply-chain.ts --n 7
```

## 出力

- `aspects/investment/candidates/YYYY-MM-DD-supply-chain.json`
- 14 日以内のファイルは次回 `/rebalance` で自動取り込み

## 結果のレビュー（厳守）

**ユーザーへの提示は最大 2 件に絞る。** スクリプトが 5 件出力してもそのまま並べない。
- Claude が信頼度・モメンタム未発火・バケット補完の観点で自分で 2 件に絞り込む
- 「どれにしますか？」と聞かない。「この 2 件を推奨する」と言い切る
- 残りは「次回候補」として一言添えるだけでよい

各候補を提示するとき：

1. **1-2 行の「未来予想図」を必ず語る**（なぜこれから動くのか、どういうシナリオで上がるのか）
2. **Thesis の 3 要素を確認する**
   - バリューチェーン上の不可欠性（独占性・切り替えコスト）
   - 構造的需要成長（なぜ 10 年増え続けるか）
   - 仮説崩壊条件
3. **1w/1m/3m モメンタムを必ず確認**（yahoo-finance2 で取得、WebSearch 禁止）
4. **推奨金額を必ず添える**（ユーザーに「いくら？」と聞かせない）
   - バケット目標とポジション上限から計算する
   - Edge Lottery: max 3% of total portfolio（例 $56K ポートフォリオなら max $1,680）
   - Diversifier Growth: 3-5%（不足バケットなら多めに）
   - 金額は `/rebalance` 本番実行時に再調整されるが、まず目安を出す

## キオクシア型の判定基準

以下をすべて満たす候補が理想：

- そのレイヤーで 2 社以下しか供給できない（寡占・独占）
- 顧客の認定プロセスに 1-3 年かかる（切り替えコスト高）
- AI/ロボット/宇宙/エネルギーの需要拡大で自動的に売上が増える
- アナリストカバー数が少ない（< 10 社）
- 時価総額 $1B-$15B

## 注意事項

- **claude の knowledge cutoff がある**。特定の非公開サプライヤー情報は不正確な可能性
- ティッカーが実在するか必ず yahoo-finance2 で検証される（存在しないティッカーは除外）
- `/discover-growth` と組み合わせて使うと、ソフト（成長株）+ ハード（川上株）の両面をカバーできる
